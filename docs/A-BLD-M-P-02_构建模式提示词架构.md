## 一、文档定位

本文定义构建模式的提示词架构，包括：

1. Master AI 提示词的分层结构。
2. Role AI 提示词框架。
3. 构建大纲生成提示词框架。
4. 与编辑模式提示词体系的隔离说明。

**本文定义结构框架，不包含具体提示词文本。** 具体文本在开发阶段细化，并在本文中补充。

---

## 二、提示词体系总览

构建模式维护一套完全独立的提示词体系，不继承或复用编辑模式（`A-AG-M-T-02`）的任何提示词。

```
构建模式提示词体系
│
├── Master AI 提示词
│   ├── 系统提示词（System Prompt）
│   │   ├── 基础角色定义
│   │   ├── 构建模式规则与约束
│   │   └── 工具调用协议说明
│   ├── 规划提示词（Planning Prompt）
│   │   └── Build Outline 生成指令
│   ├── 执行提示词（Execution Prompt）
│   │   └── 单步文件生成指令
│   └── 收束提示词（Synthesis Prompt）
│       └── 讨论收束摘要生成指令
│
└── Role AI 提示词
    ├── 系统提示词（System Prompt）
    │   ├── 角色定义注入
    │   ├── 讨论行为规范
    │   └── 禁止行为约束
    └── 任务提示词（Task Prompt）
        └── 具体讨论任务指令
```

## 三、Master AI 系统提示词框架

### 3.1 基础角色定义

> 待定：具体文本

框架要素：

1. Master AI 的核心职责声明（规划、编排、执行、收束）。
2. Master AI 的隐式存在定义（不以对话方式与用户交互）。
3. Master AI 的权限边界（可调用工具、可写入文件）。

### 3.2 构建模式规则注入

> 待定：具体文本

1. 结果导向原则（所有行为必须服务于"生成完整项目"）。

3. Project Object 结构规范（主文档+子文档+数据+资源）。
4. 文件生成质量标准（完整性、一致性、可用性）。

### 3.3 工具调用协议说明

> 待定：具体文本

框架要素：

1. 可用工具列表（参考 `A-BLD-M-P-01` 第五节）。
2. 工具调用格式要求（provider-side function calling）。
3. 工具调用失败的处理指令。

## 五、Master AI 执行提示词框架（单步文件生成）

### 5.1 输入注入结构

```
[项目上下文]

项目描述：{project_description}

[当前步骤]

文件类型：{file_type}
文件标题：{title}
内容指令：{content_instruction}
依赖文件内容：{dependency_content}
模板约束：{template_content | null}
```

### 5.2 输出指令

> 待定：具体文本

框架要素：

1. 要求输出完整的文件内容，符合指定格式（md/docx/txt）。
2. 要求与已完成文件保持一致性（风格、结构、术语）。
3. 要求不超出模板约束范围（如果有模板引用）。

## 六、Master AI 收束提示词框架（讨论收束）

### 6.1 输入注入结构

```
[讨论历史]
{recent_discussion_messages}

[当前共识]（上一版本，如有）
{previous_consensus}

[当前分歧]（上一版本，如有）
{previous_divergence}
```

### 6.2 输出指令

> 待定：具体文本

框架要素：

1. 要求输出符合 `A-BLD-M-P-01` 第四节定义的收束摘要格式。
2. 要求准确区分共识与分歧，不混淆。
3. 要求方案列表客观呈现各方观点，不偏向某一方。
4. 要求语言简洁，便于参与者快速阅读。

## 七、Role AI 系统提示词框架

### 7.1 角色定义注入

```
你是一个 AI 参与者，扮演的角色是：{role_name}

角色描述：{role_description}
```

> 待定：具体文本

### 7.2 讨论行为规范注入

> 待定：具体文本

框架要素：

1. 以指定角色的专业视角参与讨论。
2. 提供有价值的分析、建议和专业意见。
3. 简洁明了，避免冗长输出。
4. 承认不确定性，不过度自信。

### 7.3 禁止行为约束注入

> 待定：具体文本

框架要素（必须包含的禁止项）：

1. 禁止承诺执行任何写入操作（"我来帮你生成...""我可以创建..."）。
2. 禁止输出完整的文档正文（只提供建议，不替代生成）。
3. 禁止超出角色范围（只从指定角色视角发言）。
4. 禁止直接访问项目文件（只能使用 Master AI 提供的上下文）。

## 八、与编辑模式提示词的隔离

### 8.1 隔离声明

构建模式提示词体系与编辑模式提示词体系**完全隔离**，不共享任何提示词模板。

### 8.2 具体隔离边界

| 提示词类型 | 编辑模式 | 构建模式 |
|---|---|---|
| 系统提示词 | `context_manager.rs` 构建 | 构建模式独立构建 |
| 工具 schema | `tool_definitions.rs` | 构建模式独立定义 |
| 续写/局部修改提示词 | Layer 1/2 独立 | 不使用 |
| 对话编辑提示词 | Layer 3 | 不使用 |

### 8.3 可参考（不可复用）

以下编辑模式的设计思路可作为参考，但实现必须独立：

1. `A-AG-M-T-02_prompt架构.md` 中的分层设计思路。
2. `A-DE-M-P-01_对话编辑提示词.md` 中的角色定义方式。

**"参考"指设计思路可以借鉴，代码实现必须独立，不允许调用编辑模式的提示词构建函数。**
