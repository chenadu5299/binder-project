## 一、弹窗交互规范（2.2.1）

### 1.1 触发条件

- **快捷键**：Cmd+K / Ctrl+K
- **须光标激活**：编辑器聚焦时有效；**光标未激活**时快捷键无效

### 1.2 无选区调出与选中同步

- **支持无选区调出**：无选区时调出弹窗，**默认内容为空**（不取光标所在块）
- **先调出再选中**：弹窗已打开时，用户选中内容 → 窗口**实时**显示选中内容，新选中内容进入后续处理逻辑（执行时使用最新选区）
- 选中内容变化时清空已有消息，避免将旧 AI 结果应用到新选区

### 1.3 光标无效提示

- 光标未激活时按 Cmd+K：窗口底部状态栏「保存状态处」滚动显示「指令无效」等提示
- 与辅助续写（Cmd+J）的光标无效处理一致

### 1.4 弹窗内不做 Diff

- 原文档**只显示选区**（有选区时），不涉及 Diff 渲染
- **弹窗内不做 Diff 效果**：弹窗内容为对话与可插入内容，无红绿高亮

### 1.5 选区保持

- **选区不取消**：激活修改窗口后，选区应保持选中状态；不自动聚焦输入框，避免编辑器失焦导致选区取消
- **selectionRange 存储**：选区范围（from, to）在打开时存储，apply 时使用存储范围，确保替换正确

### 1.6 弹窗布局与定位

- **布局**：输入框固定在上方，内容区（选中文本 / AI 回复 / 错误）在下方；执行前后布局一致，输入框顶部稳定
- **定位**：窗口始终左右居中；垂直位置**始终在选区下方**；当选区接近文档底部、选区下方空间不足时，改为**光标上方**合适位置

### 1.7 关闭与其它

- **关闭**：点击关闭按钮、弹窗已弹出时再次 Cmd+K、点击应用 → 关闭
- 弹窗跟随窗口缩放

## 二、useInlineAssist messages 结构

### 2.1 格式

```typescript
messages: Array<{ role: "user" | "assistant"; text: string }>
```

- 仅需 `role`、`text` 两个字段
- 无对话历史持久化：每次交互覆盖上一次回复；只回复可直接插入文档的内容

### 2.2 前端职责

- useInlineAssist 维护 messages 数组
- 用户发送指令 → 追加 `{ role: "user", text: "用户输入" }`
- AI 返回 → 追加 `{ role: "assistant", text: "AI 回复" }`
- 调用 ai_inline_assist 时携带完整 messages

## 三、ai_inline_assist messages 入参格式

### 3.1 扩展后的入参

```typescript
{
  instruction: string;      // 用户指令（当前轮）
  text: string;            // 选中文本（无选区时为空；先调出再选中时实时更新）
  context: string;         // 上下文
  messages?: Array<{ role: "user" | "assistant"; text: string }>;  // 对话历史
}
```

### 3.2 后端处理

- 后端按 messages 顺序拼接为对话式提示词
- 最后追加当前 instruction + text
- 若 messages 为空或未传，按原有 (instruction, text, context) 逻辑处理

### 3.3 实现位置

- 前端：`src/hooks/useInlineAssist.ts` 收集并传递 messages
- 后端：`src-tauri/src/commands/ai_commands.rs` 中 ai_inline_assist 增加 messages 参数
