4. BLD 模块全部设计文档（A-BLD-*）完成评审。

## 三、阶段划分

### Phase 0：基础设施（前置，必须先完成）

**目标：** 为构建模式创建独立的代码结构，与编辑模式物理隔离。

**交付物：**

1. `src-tauri/src/services/build_mode/` 目录建立，包含：
   - `mod.rs`（模块声明）
   - `master_ai.rs`（Master AI 服务骨架）
   - `role_ai.rs`（Role AI 服务骨架）
   - `build_task.rs`（构建任务状态管理骨架）
   - `project_writer.rs`（Project Object 写入骨架）
2. `workspace.db` 新增 `build_tasks` 表（Schema 见 `A-BLD-I-P-01`）。
3. 前端 `src/stores/buildStore.ts` 建立（构建任务状态 Zustand Store）。
4. 构建模式路由/Tab 入口占位（UI 空壳）。

**验收：** 代码结构建立，编译通过，不影响编辑模式运行。

---

### Phase 1：直接构建主链路

**目标：** 实现直接构建路径的完整主链路（输入 → 规划 → 执行 → 产出）。

**交付物：**

1. **输入阶段 UI**：
   - 描述/需求输入表单。
   - 引用选择（Workspace 文件 + 模板库，基础版）。
   - 「开始构建」按钮。

2. **规划阶段**：
   - Master AI Build Outline 生成（`A-BLD-M-P-01` 第六节协议）。
   - Outline 展示 UI（用户确认/修改）。

3. **执行阶段**：
   - Master AI 按 Outline 顺序生成文件。
   - 每步完成后增量写入 Workspace。
   - 进度展示 UI（步骤列表 + 当前步骤高亮）。
   - 后台执行支持（Tauri 异步任务）。

4. **产出阶段**：
   - `_build_meta.json` 写入。
   - 构建完成通知。
   - Project Object 在文件树中可见。

**验收口径：**

1. 用户可完整走通：输入 → Outline 确认 → 执行 → Project Object 出现在文件树。
2. 执行期间切到编辑模式，后台继续运行，完成后通知出现。
3. 单任务并发限制生效（第二次触发被拒绝）。
4. 执行失败时已生成文件保留，`_build_meta.json` 正确记录 partial/failed 状态。

---

### Phase 2：讨论构建主链路

**目标：** 实现讨论构建路径的完整主链路。

**交付物：**

1. **讨论组创建与配置**：
   - 新建讨论组（选择类型：Open / Internal）。
   - 角色 AI 配置界面（名称、描述）。
   - 真人参与者邀请（生成分享链接）。

2. **讨论界面**：
   - 轻群聊消息流 UI。
   - 角色 AI 消息展示（带角色名称）。
   - 主控 AI 收束摘要展示（特殊样式，标注"主控 AI 整理"）。

3. **状态机实现**：
   - IN_PROGRESS / WAITING_CONFIRMATION / FINISHED 三态完整实现。
   - 状态转换按钮（继续讨论/触发收束/触发构建）。
   - FINISHED 状态下输入禁用。

4. **确认机制**：
   - 参与者「确认」按钮。
   - 确认状态展示（已确认/未确认）。
   - 触发构建时的确认快照记录。

5. **讨论结果写入**：
   - `_discussion_summary.md` 生成。
   - `_decision_snapshot.json` 生成。
   - 讨论组标签记忆写入（对接 AST）。

6. **构建触发接入 Phase 1 执行链**。

**验收口径：**

1. 用户可完整走通：创建讨论组 → 多方讨论 → 收束 → 触发构建 → Project Object 生成。
2. 状态机三态转换正确，FINISHED 不可回退。
3. 讨论结果文件正确写入 Project Object 目录。
4. 内部讨论组的项目上下文脱敏生效。

---

### Phase 3：完善与优化

**目标：** 功能完善、体验优化、边界场景处理。

**待规划内容（Phase 3 细项在 Phase 2 完成后制定）：**

> 待定：Phase 3 具体交付物

预期方向：

2. 构建历史记录查看。
3. 工作流模板引用支持（如 Phase 1/2 中未实现）。
4. 构建时长估算展示。
5. 错误提示优化。
6. 讨论组收束触发条件优化（主控 AI 智能判断）。
