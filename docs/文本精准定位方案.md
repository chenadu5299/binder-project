# 文本精准定位方案

> 整理自现有实现，保留并完善**块 + 块内偏移**（block + offset）方案。

---

## 一、方案概述

### 1.1 核心思路：块 + 块内偏移

**定位公式**：`block_id` + `start_offset` + `end_offset`

- **block_id**：块级节点的唯一标识（如 paragraph、heading 等）
- **start_offset**：目标文本在块内纯文本的起始字符下标（0-based）
- **end_offset**：目标文本在块内纯文本的结束字符下标（0-based）

**优势**：
- 块 ID 稳定，文档局部修改时其他块不受影响
- 块内偏移基于 `node.textContent`，与 HTML 标签无关
- 支持选区、引用、AI 编辑等多种场景

### 1.2 与其他方案对比

| 方案 | 定位方式 | 优点 | 缺点 |
|------|----------|------|------|
| **块 + 偏移**（当前采用） | block_id + start_offset + end_offset | 稳定、精确、跨场景 | 需块级 ID 基础设施 |
| 字符位置 | 全文 start_char / end_char | 简单 | 大文档难算、易漂移 |
| target_content 检索 | 文本匹配 + replacen | 无需位置 | 多处相同文本易替换错位 |

### 2.2 数据流

```
【场景 A：选区发送】
用户选中 → editor.selection (from, to)
  → createAnchorFromSelection(doc, from, to)
  → findBlockContainingSelection + pmRangeToBlockOffset
  → edit_target = { anchor: { block_id, start_offset, end_offset } }
  → ai_chat_stream(editTarget)
  → tool_service: extract_block_text_by_id → 单条 diff (element_identifier: JSON)
  → 前端: findBlockByBlockId + blockOffsetToPMRange → from/to → 应用修改

【场景 B：引用粘贴】
复制 → CopyReferenceExtension → createAnchorFromSelection → sourceData { blockId, startOffset, endOffset }
  → 粘贴 → TextReference 含 blockId+offset
  → 发送时 pathMatch + blockId → edit_target
  → 同场景 A 后端路径
```

## 三、块 + 偏移 方案详解

### 3.1 块内偏移约定

- **offset**：块内 `node.textContent` 的字符下标，从 0 开始
- **与 HTML 无关**：`<strong>hello</strong>` 的 textContent 为 `hello`，offset 0–4
- **跨节点**：块内可有多个 text 节点，offset 按串联后的纯文本计算

### 3.2 核心 API

```typescript
// 选区 → Anchor
createAnchorFromSelection(doc, from, to) → { blockId, startOffset, endOffset } | null

// Anchor → ProseMirror 范围
findBlockByBlockId(doc, blockId) → { node, docStart } | null
blockOffsetToPMRange(block, docStart, startOffset, endOffset) → { from, to } | null

// ProseMirror 范围 → 块内偏移
pmRangeToBlockOffset(block, blockDocStart, from, to) → { startOffset, endOffset } | null
```

### 3.3 后端 extract_block_text_by_id

- 用 `[data-block-id="{block_id}"]` 选择器定位块
- 取 `element.text()` 得到块纯文本
- 用 `chars().skip(start_offset).take(end_offset - start_offset)` 提取目标区间

## 四、待完善项

### 4.1 跨块选区

**现状**：`findBlockContainingSelection` 跨多块时只返回包含 `from` 的块。

**建议**：
- 方案 1：限制选区为单块，跨块时提示「请在同一段落内选择」
- 方案 2：支持多块 Anchor 数组，后端返回多条 diff，前端依次应用

### 4.2 无 edit_target 降级（场景 C：AI 自行编辑）

**现状**：无选区/引用时走 target_content 检索，`replacen` 第一处，易错位。

**建议**：
- 在 prompt 中注入块结构摘要，要求 AI 输出 block_id + offset
- 或：无 edit_target 时提示「请通过选中或引用指定要修改的位置」

### 4.3 路径匹配

**建议**：`isSameDocumentForEdit` 支持 workspacePath，统一相对/绝对路径比较。

### 4.4 表格、图片等复杂元素

**现状**：`element_identifier` 支持 table、image 等，与 block_id 路径并行。

**建议**：保持现有设计，block_id 用于文本块，element_identifier 用于表格/图片等。
