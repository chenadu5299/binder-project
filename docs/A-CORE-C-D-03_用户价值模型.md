## 一、文档定位

本文定义 Binder MVP 的用户价值模型，回答三件事：
1. 谁是首版核心用户  
2. 他们要完成什么关键任务  
3. Binder 如何用最小能力闭环提供可感知价值  

本文是产品与范围文档，不展开实现细节。

---

## 二、建模边界（MVP）

1. 平台边界：仅 macOS。  
2. 能力边界：文件管理 + 文档编辑 + AI 三层 + 对话编辑 Diff 主链。  
3. 开源边界：仅开源核心链路；预览子系统、T-DOCX、分页不纳入开源首版能力承诺。  
4. 模型边界：官方适配 DeepSeek / OpenAI / Anthropic，并保留 OpenAI 兼容接入。  

---

## 三、核心用户分层

### 3.1 A 层：高频文档生产者（首要目标用户）

用户画像：
- 长文写作者、技术文档作者、产品文档作者、研究型写作者  
- 日常高频“写-改-重构-定稿”  

核心痛点：
- 写作和修改割裂，频繁在编辑器与外部 AI 工具切换  
- AI 改写不可控，改错位置或整文覆盖风险高  
- 多轮修改后难追溯“改了什么、为何失效”  

Binder 价值：
- 在同一工作区完成“编辑 + 对话修改 + 人工确认”  
- 通过 Diff 卡把 AI 改动变成可决策对象  
- 减少黑盒改写，提升可控性与信任感  

### 3.2 B 层：项目型知识工作者（次目标用户）

用户画像：
- 需要在多个文档间来回整理内容的产品/运营/咨询人员  

核心痛点：
- 跨文档修改路径断裂，目标文件易混淆  
- 引用资料与实际修改脱节  

Binder 价值：
- 以 Workspace 为统一上下文容器  
- 对话编辑支持“按目标文件执行”，降低错改概率  
- 引用系统让上下文进入同一编辑主链  

### 3.3 C 层：轻量 AI 使用者（覆盖但非优化重点）

用户画像：
- 偶尔使用 AI 辅助润色、续写、局部改写  

核心痛点：
- AI 功能入口复杂，不知道何时该用哪种能力  

Binder 价值：
- 三层 AI 分工明确（续写/局改/对话编辑）  
- 新手可先用层次一、层次二，逐步过渡到层次三

## 四、用户任务模型（JTBD）

### 4.1 主任务 T1：高质量完成文档内容生产

用户要完成：
1. 快速起草内容  
2. 针对局部进行精修  
3. 对多处修改进行可控合并  
4. 在可回看的过程里完成定稿  

成功判定：
- 修改结果与意图一致  
- 用户能理解每次变更并决定是否生效  

### 4.2 主任务 T2：在同一工作区内连续推进任务

用户要完成：
1. 管理文件与目录  
2. 在文档和聊天间连续操作不中断  
3. 在引用上下文下完成针对性编辑  

成功判定：
- 不需要反复切换外部工具  
- 目标文件明确、执行路径清晰  

### 4.3 主任务 T3：把 AI 变成“可控编辑器”而不是“黑盒生成器”

用户要完成：
1. 明确 AI 改了哪里  
2. 对每条修改独立决策  
3. 失败和失效能区分，并可继续完成其余任务  

成功判定：
- 不因单点失败中断整轮任务  
- 用户对状态变化有稳定预期

## 五、价值闭环（用户视角）

### 5.1 闭环链路

资源组织 -> 文档编辑 -> AI 生成建议 -> Diff 决策 -> 内容生效 -> 下一轮继续编辑

### 5.2 每一环的用户价值

1. 资源组织：减少“找文件、找上下文”的时间成本。  
2. 文档编辑：保证基础写作体验稳定可用。  
3. AI 建议：提供增量价值，不直接覆盖用户成果。  
4. Diff 决策：把风险暴露为可操作对象。  
5. 内容生效：仅在用户确认后改变真实内容。  
6. 下一轮编辑：保持连续性，不因历史建议混乱。

## 六、MVP 价值验收指标（产品侧）

### 6.1 任务完成指标

1. 主链完成率：用户可从“发起修改”走到“确认生效”并完成保存。  
2. 单次编辑可控率：用户能明确看到并决策每条修改。  
3. 多轮可持续率：多轮后仍可继续稳定生成和处理 Diff。  

### 6.2 效率指标

1. 首次可用速度：新用户可在短时间内完成一次完整链路。  
2. 操作连续性：主流程中无高频中断点（无意义弹窗、无定位漂移导致反复重试）。  

### 6.3 信任指标

1. 错改感知率下降：用户对“AI改错位置”的投诉明显下降。  
2. 黑盒感下降：用户可解释“为何生效/为何失效/为何跳过”。

## 七、非目标用户与非目标价值（MVP 不承诺）

1. 需要跨平台一致体验的团队级用户（Windows/Linux 首发不承诺）。  
2. 以演示文稿/电子表格为核心生产对象的用户。  
3. 强依赖分页排版、预览引擎、复杂文档出版级能力的用户。  
4. 期望“全自动无人确认改稿”的用户。

## 九、版本结论（MVP）

Binder MVP 的核心价值不是“让 AI 自动写完文档”，而是：
在本地工作区内，把文档生产过程变成一条连续、可控、可确认、可追踪的编辑闭环。
