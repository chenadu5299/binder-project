# 326_excel能力

## 一、文档定位

本文是编辑器系统中 Excel 能力的统一主规范，收口 Excel 的预览、编辑、公式、多工作表与保存回写主链。  
本文用于整合需求、开发方案与技术验证结论，不替代验证明细文档。

---

## 二、MVP 目标

1. 提供稳定的 Excel 预览与编辑双模式能力。  
2. 保证数据与公式完整性，避免“值替代公式”等不可逆丢失。  
3. 建立从需求到实现再到验证的统一口径，减少文档分散。  

---

## 三、能力范围（MVP）

## 3.1 预览能力

1. Excel 文件只读预览可用。  
2. 支持基础交互：查看、搜索、缩放、复制、打印。  
3. 支持多工作表预览入口与页面导航语义。  

## 3.2 编辑能力

1. 单元格编辑、行列操作、基础格式编辑。  
2. 基础公式输入、解析、计算与保留。  
3. 多工作表切换与基础管理。  
4. 自动保存与手动保存主链。  

## 3.3 不在 MVP 范围

1. 全量高级公式库覆盖。  
2. 复杂 BI 级分析功能。  
3. 跨文档实时联动计算。  

---

## 四、统一架构（收口）

## 4.1 主链分层

1. 预览链  
Excel -> 转换/解析 -> 预览渲染。  

2. 编辑链  
Excel -> 中间表示 -> 编辑器表格模型。  

3. 保存链  
编辑器表格模型 -> 中间表示 -> Excel 回写。  

4. 校验链  
公式、格式、多工作表一致性校验 -> 失败暴露。  

## 4.2 核心原则

1. 预览链与编辑链分离，职责不混用。  
2. 公式是一等对象，必须可保留、可回写、可验证。  
3. 数据准确性优先于样式完整度。  

---

## 五、关键规则

## 5.1 公式规则

1. 公式在解析、编辑、保存全过程保持表达式语义。  
2. 禁止将公式静默固化为计算结果后再保存。  
3. 公式错误需可观测并可定位到单元格。  

## 5.2 多工作表规则

1. 工作表是独立数据域，切换不丢状态。  
2. 跨表引用需保持引用关系语义。  
3. 工作表管理操作必须进入统一保存链。  

## 5.3 格式规则

1. 基础格式（字体、对齐、边框、数字格式）为 MVP 必须能力。  
2. 样式处理不得破坏数据与公式。  
3. 格式超出支持范围时，优先保证可编辑与可回写。  

## 5.4 保存与一致性规则

1. 自动保存与手动保存共享同一写入策略。  
2. 保存失败必须显式暴露，禁止静默吞错。  
3. 回写后需具备可校验闭环（抽检公式/工作表/格式关键点）。  

---

## 六、验证闭环（由验证文档上收）

1. P0 验证：公式保留、公式写回、数据不丢失。  
2. P1 验证：多工作表、合并单元格、数字格式、跨表引用。  
3. P2 验证：性能与大文件稳定性。  
4. 验证失败项必须回写到本主文档的“限制与风险”清单。  

---

## 七、与其他模块协作边界

1. 与 `321_编辑器架构`：Excel 能力是编辑器子系统，不接管主事务。  
2. 与 `202_workspace架构`：文件打开、缓存、落盘由 Workspace 主链治理。  
3. 与 `203_数据流状态流`：Excel 编辑状态进入统一状态流与观测流。  
4. 与 `204_ai执行架构`：AI 工具操作表格时必须遵循公式/保存一致性规则。  

---

## 八、MVP 验收口径

1. 预览、编辑、保存三链稳定可用。  
2. 公式在编辑前后保持可追踪与可回写。  
3. 多工作表主链可用，关键操作不丢数据。  
4. 关键失败场景有统一暴露与回退策略。
