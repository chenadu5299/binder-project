## 二、层次一：辅助续写

### 2.1 统一展示方式

层次一不做 diff。

统一使用：

- 光标位置悬浮卡
- 一次返回多条候选建议
- 用户选择一条后直接写入文档

### 2.2 禁止做法

禁止：

- 幽灵文字混排
- 文档正文内联红绿 diff
- 在原文档中保留待确认删除标记

## 三、层次二：局部修改

### 3.1 统一展示方式

层次二不做对话编辑式 diff。

统一使用：

- 局部修改弹窗
- 弹窗内多轮对话
- 原文档仅保留选区感知

### 3.2 禁止做法

禁止：

- 在原文档中展示层次三那套删除标记
- 在弹窗内复用层次三 diff 卡池

## 四、层次三：对话编辑

### 4.1 文档侧展示

文档侧只承担删除标记展示。

规则：

- 待确认 diff 在文档侧显示删除标记
- 文档侧不直接插入新增内容预览
- diff 失效时，删除标记直接消失

### 4.2 聊天侧展示

聊天侧承担完整 diff 卡展示。

卡片统一分为三态：

1. 待执行
2. 已失效
3. 已执行

展示规则以 [R-DE-M-R-02_对话编辑-统一整合方案.md](/Users/chenzhenqiang/Desktop/test/binder-project/docs/R-DE-M-R-02_对话编辑-统一整合方案.md) 第九节为准。

### 4.3 标题展示

统一口径：

- 显示路径
- 显示标题 + 块号位置
- `document_level` 仅显示标题
- 不使用行号作为定位口径

### 4.4 交互口径

层次三允许：

- 单卡接受
- 单卡拒绝
- 全部接受
- 全部拒绝
- 已执行卡折叠/展开

层次三不允许：

- 失效提示 toast
- 失效原因弹窗
- diff 系统自身回撤

### 4.5 静默失效

当 diff 失效时：

- 文档侧删除标记消失
- 聊天侧卡片转为灰色历史态
- 不弹提示
- 不做额外打断

## 五、层次三的视觉一致性要求

### 5.1 待执行卡

显示：

- 路径
- 标题 + 位置
- 完整内容
- 操作键

### 5.2 失效卡

显示：

- 路径
- 标题 + 位置
- 完整内容
- 灰色展示
- 不折叠

### 5.3 已执行卡

显示：

- 路径
- 标题 + 位置
- 默认折叠
- 可展开查看完整内容
