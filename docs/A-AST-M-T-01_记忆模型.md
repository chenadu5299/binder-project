## 一、文档定位

本文是记忆上下文系统中的记忆模型统一主规范，收口记忆分层、作用域、数据模型、生命周期与检索协议。  
本文承接需求锚定与主控口径，补齐模型定义层并增强与全局架构映射。

本文只回答：

1. 记忆有哪些层。
2. 每层记忆存什么、如何生灭、如何检索。
3. 记忆项的数据模型如何统一。

本文不直接回答：

1. 当前轮 Agent artifact 是否应进入记忆。
2. 对话记忆与当前轮状态如何分工。
3. 哪些中间态可以升格为项目 / 工作区 / 用户记忆。

这些问题由 `A-AST-M-D-01_Binder Agent记忆协同主控文档.md` 负责。

## 二、MVP 目标

1. 建立可执行的四层记忆模型（标签级、项目内容、工作区长期、用户级）。  
2. 建立记忆写入、更新、删除、检索的统一语义。  
3. 建立记忆模型与 AI 执行架构、Workspace 架构、引用系统的映射关系。  

---

## 三、模型总览

## 3.1 记忆层级

1. 标签级记忆（`tab`）  
作用域：单聊天标签；生命周期：随标签删除。  

2. 项目内容记忆（`content`）  
作用域：工作区项目语义层；生命周期：随文档变化更新。  

3. 工作区长期记忆（`workspace_long_term`）  
作用域：工作区；生命周期：只追加不回写修改。  

4. 用户级记忆（`user`）  
作用域：用户；生命周期：跨项目继承并演化。  

## 3.2 设计原则

1. 显式引用优先于自动检索。  
2. 作用域隔离优先于召回规模。  
3. 先结构化存储，再语义增强。  
4. 记忆写入失败不阻塞主对话。  
5. 当前轮 Agent 状态与 artifact 默认不直接纳入本文定义的记忆层。

## 四、核心数据模型

## 4.1 记忆项（MemoryItem）

最小字段集合：
1. `id`  
2. `layer`  
3. `scope_type`  
4. `scope_id`  
5. `entity_type`  
6. `entity_name`  
7. `content`  
8. `summary`  
9. `source_kind`  
10. `source_ref`  
11. `confidence`  
12. `freshness_status`  
13. `readonly`  
14. `created_at`  
15. `updated_at`

### 4.1.1 字段完整语义契约

| 字段 | 类型 | 必填 | 语义说明 | 约束 |
|------|------|------|---------|------|
| `id` | TEXT (UUID v4) | ✅ | 全局唯一标识 | 写入后不可修改 |
| `layer` | ENUM | ✅ | 记忆层级：`tab`/`content`/`workspace_long_term`/`user` | 写入后不可修改 |
| `scope_type` | ENUM | ✅ | 作用域类型：`tab`/`workspace`/`user` | 与 layer 对应，见 §4.1.2 |
| `scope_id` | TEXT | ✅ | 作用域主键：tab_id / workspace_path / user_id | 写入后不可修改 |
| `entity_type` | TEXT | ✅ | 实体类型（见 A-AST-M-S-01 §2.4） | 非空 |
| `entity_name` | TEXT | ✅ | 实体名称（唯一性由 scope_id+entity_name+layer 组合保证） | 非空；用于 upsert 去重 |
| `content` | TEXT | ✅ | 完整记忆内容 | 非空；注入时使用 summary，不直接注入 content |
| `summary` | TEXT | ✅ | 15 字以内的简短摘要（直接用于注入） | 可为空字符串（退化为 content 前 100 字） |
| `tags` | TEXT | ✅ | 空格分隔的关键词（供 FTS5 索引） | 可为空字符串 |
| `source_kind` | ENUM | ✅ | 来源类型（7 种，见 §4.2） | 写入时必须明确 |
| `source_ref` | TEXT | ✅ | 来源引用路径（tab_id / file_path / 对话轮次 ID） | 可为空字符串，但有值时便于溯源 |
| `confidence` | REAL [0.0-1.0] | ✅ | AI 提炼时的置信度 | 默认 0.8；手动写入时为 1.0 |
| `freshness_status` | ENUM | ✅ | 新鲜度：`fresh`/`stale`/`expired`/`superseded` | 写入时默认 fresh；见 §4.1.3 |
| `readonly` | BOOL | ✅ | 是否只读（系统生成的记忆为 1） | 默认 1；P0 所有记忆均为只读 |
| `access_count` | INT | - | 被注入次数（影响排序权重） | 默认 0；每次注入后 +1 |
| `last_accessed_at` | INT (Unix秒) | - | 最近一次被注入的时间 | 可为 NULL |
| `created_at` | INT (Unix秒) | ✅ | 创建时间 | 写入后不可修改 |
| `updated_at` | INT (Unix秒) | ✅ | 最后更新时间 | upsert 时更新 |

### 4.1.2 layer 与 scope_type 的对应约束

| layer | scope_type | scope_id 含义 |
|-------|-----------|--------------|
| `tab` | `tab` | chatStore 的 tab UUID |
| `content` | `workspace` | workspace 绝对路径 |
| `workspace_long_term` | `workspace` | workspace 绝对路径 |
| `user` | `user` | Tauri app 唯一用户 ID |

违反此约束的写入请求必须被拒绝（不写入，记录 warn 日志）。

### 4.1.3 freshness_status 行为定义

| 状态 | 触发条件 | 在检索中的行为 | 注入时的处理 |
|------|---------|--------------|------------|
| `fresh` | 写入时默认；7 天内更新过 | 正常权重参与检索 | 正常注入 |
| `stale` | 超过 7 天未更新（P1 定期任务） | 排序降权 0.5x | 注入时加注"（信息可能已过时）" |
| `expired` | 标签删除 / 文档删除 | 不参与自动检索 | 不注入（仅 UI 可见） |
| `superseded` | 同 scope_id+entity_name 有更新版本 | 不参与自动检索 | 不注入（内部保留） |

**工程落地文档**：字段完整 SQL 定义见 `A-AST-M-S-01_记忆服务数据库落地规范.md` §二。

## 4.2 来源模型（MemorySource）

冻结来源类型：
1. `conversation_summary`  
2. `conversation_entity`  
3. `document_extract`  
4. `document_outline`  
5. `document_detail_enrichment`  
6. `tab_deletion_summary`  
7. `user_preference`

## 4.3 关系模型（MemoryEdge）

1. 记忆项之间可建立引用/归属/演化关系。  
2. 关系必须可追溯来源，不允许隐式推断直接落库。

## 五、生命周期模型

## 5.1 生成链

1. 对话触发 -> 标签级记忆抽取。  
2. 文档触发 -> 项目内容记忆抽取。  
3. 标签删除 -> 删除行为摘要写入长期记忆。  
4. 跨项目行为 -> 用户级记忆增量演化。  

## 5.2 更新链

1. 标签级记忆：仅在标签存续期内更新。  
2. 项目内容记忆：随文档结构与内容变化更新。  
3. 长期记忆：只追加，不回写修改。  
4. 用户级记忆：可演化，不受项目切换影响。  

## 5.3 删除链

1. 删除标签 -> 标签级记忆删除。  
2. 删除标签不会回写删除长期记忆历史。  
3. 工作区切换不会携带前工作区标签/项目记忆。  

---

## 六、检索协议

## 6.1 检索顺序

`显式引用 > 标签级记忆 > 项目内容记忆 > 长期记忆 > 用户级记忆`

补充说明：

1. 本顺序只定义**记忆层内部**的检索顺序。
2. 当前轮 Agent 状态、artifact、当前文档/选区等更高优先级对象，由 `A-AST-M-P-01_上下文注入.md` 与 `A-AST-M-D-01_Binder Agent记忆协同主控文档.md` 定义，不在本文内部顺序中混写。

## 6.2 检索范围

`tab | content | workspace_long_term | user | all`

## 6.3 检索规则

1. 先作用域过滤，再相关性排序。  
2. 同层优先最近更新、置信度高、来源明确项。  
3. 检索失败或超时按空结果降级，不阻塞主链。

## 八、约束与非目标

1. 记忆库不是用户可自由编辑的笔记系统。  
2. 首版不引入外部向量数据库作为主存。  
3. 不在层次一/层次二默认接入记忆模型。  
4. 不以召回数量替代作用域正确性。  

---

## 九、MVP 验收口径

1. 四层记忆对象可区分、可检索、可追溯。  
2. 标签删除、项目切换、跨项目继承行为符合模型定义。  
3. 检索顺序、范围、降级规则可复现。  
4. 与 `A-WS-M-T-01_workspace架构.md`/`A-SYS-M-T-01_数据流状态流.md`/`A-AG-M-T-01_ai执行架构.md`/`A-AG-M-T-02_prompt架构.md`/`A-AST-M-P-01_上下文注入.md` 文档口径一致。
