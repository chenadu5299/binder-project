## 一、文档定位

本文是验证与运维层的测试矩阵主规范，定义 Binder MVP 的测试维度、分层用例、优先级与验收门槛。  
本文承接测试指南与 Excel 专项验证方案，补齐矩阵化测试框架。

---

## 二、MVP 目标

1. 建立覆盖产品主链的统一测试矩阵。  
2. 建立分级优先级（P0/P1/P2）与准入门槛。  
3. 建立通用测试流与专项验证流的对齐机制。  

---

## 三、测试矩阵维度

## 3.1 业务链维度

1. 工作区与文件主链（打开、创建、保存、刷新）。  
2. 编辑器主链（编辑、格式、自动保存、预览）。  
3. AI 三层主链（续写、局改、对话编辑）。  
4. Diff 主链（生成、展示、接受、拒绝、失效）。  
5. 资源与跨文档主链（引用、检索、跨文档操作）。  

## 3.2 技术层维度

1. 前端交互层。  
2. 后端命令层。  
3. 工具调用层。  
4. 流式响应与状态同步层。  
5. 数据一致性与恢复层。  

## 3.3 文件类型维度

1. md/txt/html。  
2. docx。  
3. xlsx/ods（Excel 能力专项）。  
4. 媒体预览相关格式（只测预览链）。  

---

## 四、优先级分级

## 4.1 P0（阻塞发布）

1. 工作区打开与文件读写稳定。  
2. AI 三层入口可用且无阻塞崩溃。  
3. 对话编辑 diff 生成与单卡接受可用。  
4. 基础错误可见（请求失败、工具失败、流中断）。  

## 4.2 P1（影响核心体验）

1. 批量接受/拒绝流程。  
2. 跨文档引用与上下文注入稳定性。  
3. docx 编辑链稳定性。  
4. Excel 核心转换链验证通过。  

## 4.3 P2（优化项）

1. 大文件性能与长会话稳定性。  
2. 极端输入与边界格式兼容。  
3. 回归自动化覆盖率提升。

## 六、专项测试并入规则（Excel）

1. Excel 专项验证结果纳入统一矩阵，不作为孤立文档。  
2. Excel P0 用例至少覆盖：公式保留、ODS 解析、ODS 生成。  
3. Excel P1 用例覆盖：多工作表、合并单元格、跨表引用。  
4. 专项失败需回写到总矩阵并阻断对应发布门槛。  

---

## 七、执行与记录规范

1. 每条用例记录：版本、环境、步骤、结果、日志链接。  
2. 失败必须带复现路径与最小重现条件。  
3. 阻塞项必须标记归属模块与责任人。  
4. 回归测试按发布节奏至少执行一次 P0 全量。  

---

## 八、发布准入门槛（MVP）

1. P0 用例 100% 通过。  
2. P1 用例通过率不低于 90%，且无高风险未决缺陷。  
3. 核心链路无 P0/P1 回归。  
4. 所有阻塞缺陷有关闭记录。
