## 一、文档定位

本文定义 Binder MVP 的标准化使用场景体系。  
目标：统一“产品主场景 + AI 子场景”的层级、编号、边界和验收口径。

本文是场景总览，不替代功能规格与实现方案。

---

## 二、场景层级标准（统一口径）

为避免场景描述粒度混乱，统一采用四层：

1. **L0 主路径场景**：端到端用户任务闭环（跨模块）。  
2. **L1 领域场景**：单领域完整任务（如文件管理、对话编辑）。  
3. **L2 功能子场景**：可独立测试的功能片段（如自动保存、引用拖拽）。  
4. **L3 交互动作场景**：具体交互步骤（如 Cmd+J、点“全部接受”）。  

编号规则：
- L0：`S0-x`
- L1：`S1-x`
- L2：`S2-x.y`
- L3：`S3-x.y.z`

---

## 三、L0 主路径场景（MVP）

## S0-1 本地文档生产主链

路径：进入工作区 -> 打开/新建文件 -> 编辑 -> 保存 -> 回读验证  
成功标准：文档内容可持续编辑，保存一致，状态可解释。

## S0-2 AI 辅助编辑主链

路径：触发 AI（层次一/二/三）-> 获得建议 -> 用户决策 -> 文档生效  
成功标准：AI 结果可控，不出现未确认直接写入。

## S0-3 对话编辑可确认主链

路径：对话请求 -> 生成 diff -> 卡片决策 -> 应用/失效 -> 下一轮继续  
成功标准：pending/accepted/expired 语义稳定，链路可追踪。

## S0-4 Workspace 连续工作主链

路径：资源管理 -> 文档切换 -> 引用上下文 -> 多轮任务持续推进  
成功标准：工作区状态一致，不因切换导致上下文错乱。

## 四、L1 领域场景

## S1-1 文件与资源管理场景

包含：
- 工作区打开/切换
- 文件树浏览与操作
- 文件导入、重命名、删除、移动
- 资源区（记忆/历史）基础交互

## S1-2 文档编辑场景

包含：
- 富文本编辑
- Markdown 编辑
- docx 预览/编辑切换（MVP 范围内）
- 自动保存、手动保存、外部修改处理

## S1-3 AI 三层场景

包含：
- 层次一：辅助续写（快捷键触发，悬浮建议）
- 层次二：局部修改（弹窗多轮）
- 层次三：对话编辑（Agent/Chat、工具调用、diff 决策）

## S1-4 引用与上下文场景

包含：
- 文本/文件/文件夹/链接等引用
- @ 选择、拖拽、粘贴
- 引用降级为普通文本
- 引用内容注入 AI 上下文

## S1-5 可观测与恢复场景

包含：
- 流式响应异常处理
- 工具调用失败分级
- 局部失败跳过继续
- 关键事件可追踪与复现

## 五、L2 功能子场景清单（MVP 优先）

## S2-1 文件管理子场景

- `S2-1.1` 打开工作区  
- `S2-1.2` 新建文件/文件夹  
- `S2-1.3` 导入文件并打开  
- `S2-1.4` 文件树操作（重命名/删除/移动）  
- `S2-1.5` 工作区切换后状态恢复  

## S2-2 文档编辑子场景

- `S2-2.1` 富文本基础编辑  
- `S2-2.2` 自动保存与手动保存  
- `S2-2.3` 多标签页编辑状态管理  
- `S2-2.4` 外部修改检测与处理  
- `S2-2.5` docx 可编辑主链（不含分页/T-DOCX）  

## S2-3 AI 三层子场景

- `S2-3.1` 辅助续写（Cmd+J）  
- `S2-3.2` 局部修改（Cmd+K）  
- `S2-3.3` 对话编辑 Agent 模式  
- `S2-3.4` 对话编辑 Chat 模式（纯聊天）  
- `S2-3.5` 工具调用与结果回流  

## S2-4 Diff 子场景

- `S2-4.1` diff 生成与展示  
- `S2-4.2` 单卡接受/拒绝  
- `S2-4.3` 全部接受/全部拒绝  
- `S2-4.4` 失效静默处理  
- `S2-4.5` 多轮 diff 持续可执行  

## S2-5 引用子场景

- `S2-5.1` 文本引用（含定位信息）  
- `S2-5.2` 文件/文件夹引用  
- `S2-5.3` 链接自动识别  
- `S2-5.4` 引用标签展示与管理  
- `S2-5.5` 引用降级（删除标记后变普通文本）  

## S2-6 可观测子场景

- `S2-6.1` 流式重复/错序检测  
- `S2-6.2` tool_call 参数异常处理  
- `S2-6.3` 错误分级（Retryable/Skippable/Fatal）  
- `S2-6.4` 跳过继续与批量汇总  
- `S2-6.5` 最小排障闭环可复现

## 六、MVP 场景边界

## 6.1 必须覆盖

1. S0-1、S0-2、S0-3、S0-4 全部可跑通。  
2. L2 中 `S2-1` 到 `S2-6` 至少覆盖每组关键子场景。  
3. 对话编辑相关场景必须满足“先确认再生效”。  

## 6.2 暂不纳入

1. Excel/演示文稿完整编辑场景。  
2. 移动端交互场景。  
3. 构建模式全流程场景。  
4. 分页/T-DOCX 相关场景。
