## 一、文档定位

本文是 Binder Agent 的**文档生成流程**统一主规范，定义从用户生成意图到文档落地交付的标准链路。

本文只回答：

1. Agent 如何将工作流模板、知识、记忆、工具整合进文档生成主链。
2. 生成、审阅、写入、导出的流程顺序如何成立。
3. 生成流程与编辑模式、构建模式的边界如何区分。

本文不直接回答：

1. 模板库对象模型与调用机制（由 `A-TMP-M-T-01_模板机制.md` 负责）。
2. 工作流模板在 Agent 体系中为何属于用户主导约束资产（由 `A-TMP-M-D-01_Binder Agent模板协同主控文档.md` 负责）。
3. 上下文预算裁剪与降级规则（由 `A-AST-M-P-01_上下文注入.md` 负责）。

---

## 二、MVP 目标

1. 建立统一的文档生成主链，覆盖输入、规划、生成、审阅、写入、导出。
2. 建立编辑模式与对话编辑场景下的可执行生成流程。
3. 建立工作流模板、记忆、知识、工具调用在生成流程中的统一接入点。

---

## 三、流程总览

标准流程：

1. 任务输入归一化
2. 生成计划构建
3. 上下文与资源注入
4. 内容生成执行
5. 结果审阅与修订
6. 文档写入与格式收敛
7. 导出与结果回流

---

## 四、标准流程定义

## 4.1 任务输入归一化

输入来源：

1. 用户自然语言目标。
2. 用户显式引用（文档、片段、模板、资源）。
3. 当前工作区与当前文档状态。

输出：

1. `generation_goal`
2. `constraint_set`（格式、长度、风格、范围）
3. `target_artifact`（目标文档类型与交付形式）

## 4.2 生成计划构建

1. 将目标拆分为可执行阶段（大纲、草稿、修订、定稿）。
2. 识别是否需要工作流模板。
3. 识别是否需要工具调用（读文件、检索、编辑写入等）。
4. 若使用工作流模板，则在进入执行前完成结构解析与流程编译。

输出：

1. `plan_steps`
2. `required_sources`
3. `execution_mode`（直接生成 / 分步生成）

## 4.3 上下文与资源注入

注入顺序固定：

`用户引用 > 当前文档/选区 > 当前轮 Agent 状态 / artifact > 已选模板资产 > 工作区相关 > 记忆库 > 知识库 > 聊天历史`

规则：

1. 显式引用优先，不允许被自动检索覆盖。
2. 当前轮 Agent 状态 / artifact 优先于自动记忆与知识检索。
3. 工作流模板作为过程约束输入，不作为最终内容替代。
4. 注入失败时降级为最小可执行上下文，不阻塞主链。

注入预算与裁剪规则以 `A-AST-M-P-01_上下文注入.md` 为准。

## 4.4 内容生成执行

1. 按计划步骤逐段生成，避免一次性长文本失控。
2. 生成阶段消费的是工作流模板编译后的结构化流程表示，而不是原始模板文本。
3. 对需要高准确性的内容先检索后生成。

输出：

1. `draft_segments`
2. `structured_sections`
3. `generation_log`

## 4.5 结果审阅与修订

1. 审阅结构完整性（章节是否齐全）。
2. 审阅约束一致性（格式、语气、目标是否匹配）。
3. 必要时进入二次修订循环。

输出：

1. `final_content`
2. `revision_notes`

## 4.6 文档写入与格式收敛

1. 通过编辑器写入链执行最终写入。
2. 保持格式协议一致（字体白名单、字号、颜色、结构）。
3. 写入失败时保留可恢复中间结果。

## 4.7 导出与结果回流

1. 按目标格式导出（如 docx / pdf / md）。
2. 生成结果回流到会话与工作区对象。
3. 可选写入记忆与结果记录。

## 五、两种执行路径

## 5.1 编辑模式路径（MVP 主路径）

1. 用户在对话编辑中发起生成请求。
2. Agent 在同一会话内完成规划、生成、修订。
3. 结果通过编辑器写入与 Diff 能力交付。

## 5.2 构建模式路径

构建模式（BLD 模块）拥有**独立的执行引擎**（`A-BLD-M-T-04_构建执行引擎.md`），不接入本文的七步生成流程框架。

本文的七步流程仅适用于编辑模式场景。

构建模式与模板库的接口关系见 `A-BLD-I-P-02_构建模式与模板库接口.md`。

---

## 六、工作流模板接入规则

1. 工作流模板只控制步骤序列、任务顺序与过程依赖。
2. 工作流模板不承担输出结构、风格偏好与行为方法定义。
3. 工作流模板不替代知识、记忆与当前文档事实。
4. 工作流模板在进入执行前，必须先完成结构解析与流程编译。
5. 生成链使用的是编译后的结构化流程表示，而不是原始模板文本。

优先级：

1. 用户显式选用模板 > 系统自动建议模板。
2. 若模板冲突，用户显式选择优先。

模板对象模型与调用机制以 `A-TMP-M-T-01_模板机制.md` 为准，本文不重复定义。

---

## 七、失败降级与暴露

1. 资源不足：降级为最小上下文生成，记录 `E_GEN_CONTEXT_INSUFFICIENT`。
2. 计划失败：降级为直接生成，记录 `E_GEN_PLAN_FAILED`。
3. 写入失败：保留文本结果并提示可重试写入，记录 `E_GEN_APPLY_FAILED`。
4. 导出失败：保留编辑器内结果，记录 `E_GEN_EXPORT_FAILED`。

---

## 八、验收口径（MVP）

1. 生成流程可按七步稳定执行并可复现。
2. 工作流模板、上下文、工具调用接入点一致。
3. 失败降级不阻塞任务完成。
4. 结果可写入文档并可在会话中追溯来源。
