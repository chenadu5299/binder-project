# 324_docx能力

## 一、文档定位

本文是编辑器系统中 DOCX 能力的统一主规范，收口 DOCX 的预览、编辑、样式处理、保存回写与边界约束。  
本文作为 321/323 的 DOCX 子系统文档，不替代历史专项说明文档。

---

## 二、MVP 目标

1. 提供稳定的 DOCX 打开链：可预览、可编辑、可保存。  
2. 明确预览链与编辑链分离，避免混用导致状态错乱。  
3. 在可编辑能力范围内保证结构与基础样式稳定，不承诺复杂 Office 格式全保真。  

---

## 三、能力范围与边界

## 3.1 能力范围（MVP）

1. 预览模式：DOCX 转 PDF 预览，支持浏览与基础交互。  
2. 编辑模式：DOCX 转 HTML 进入编辑器，支持结构化编辑与保存回写。  
3. 样式范围：标题、段落对齐、基础文字样式、列表、基础表格、图片。  
4. 分页协作：编辑态可接入分页系统，分页仅影响显示不改变内容真源。  

## 3.2 不在 MVP 范围

1. 页眉页脚精确保真编辑。  
2. SmartArt、VBA、复杂文本框布局。  
3. 复杂表格格式的完整保真（合并策略/复杂边框语义）。  
4. 与原生 Word 完全一致的版式还原。  

---

## 四、统一架构（收口）

## 4.1 三链分离

1. 预览链  
DOCX -> 转换服务 -> PDF -> 预览组件。  

2. 编辑链  
DOCX -> 转换服务 -> HTML -> 编辑器。  

3. 保存链  
编辑器 HTML -> 转换服务 -> DOCX 回写。  

## 4.2 核心原则

1. 三条链路职责分离，不互相替代。  
2. 预览模式不是编辑模式的弱化版，编辑模式不是预览模式的扩展版。  
3. 所有写回动作统一进入保存链，不允许绕过。  

---

## 五、文件来源与模式策略

1. Binder 新建/AI 生成 DOCX：可直接进入编辑链。  
2. 外部导入 DOCX：默认先进入预览链，再按规则转入编辑链。  
3. 草稿副本策略：外部文件进入编辑时可采用草稿副本隔离原件风险。  
4. 来源识别与模式切换规则必须由统一策略层决定，不由视图层临时判断。  

---

## 六、样式处理统一口径

## 6.1 样式处理分层

1. 提取层  
提取稳定可复现样式与结构。  

2. 结构层  
保证段落/列表/表格/图片结构完整可编辑。  

3. 工具层  
用户可通过工具栏继续编辑样式，不覆盖结构语义。  

## 6.2 样式边界原则

1. 优先结构稳定，其次样式精细度。  
2. 复杂样式超出支持范围时，保留内容与可编辑性优先。  
3. 禁止在样式修复中破坏文档结构或保存可逆性。  

---

## 七、与分页系统协作（323 对齐）

1. DOCX 编辑态可启用分页模式。  
2. 分页属于显示增强，不改变 DOCX 编辑链的内容语义。  
3. 分页容器、缩放、光标遮罩的改动不得影响 DOCX 保存链。  
4. DOCX 专项分页优化必须先满足 323 的统一分页规则。  

---

## 八、与编辑器主干协作（321 对齐）

1. DOCX 能力是文件适配子系统，不接管编辑器主事务。  
2. 打开/编辑/保存必须复用编辑器主链，不另建并行写入路径。  
3. 与 Block/Diff/AI 协作时，统一使用编辑器语义坐标与状态链。  

---

## 九、MVP 验收口径

1. 外部导入 DOCX 能稳定预览并可进入编辑。  
2. 编辑后可稳定保存回写 DOCX。  
3. 基础样式与结构保持可读、可编辑、可回写。  
4. 与 323 分页系统协作时不出现内容真源漂移。
