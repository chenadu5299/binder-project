> 版本：v1.1 | 创建日期：2026年3月 | 更新：1.2 @ 列表实现与优化记录
> 
> 依据：《引用功能完整设计文档》、AI与其他功能交互规范

> 原则：以设计文档为准；旧逻辑与设计文档冲突时，采用设计文档方案；不保留旧逻辑的降级/兼容计划

## 一、当前实现与目标态对比

### 1.1 数据流对比

| 环节 | 当前实现 | 目标态（设计文档） |
|------|----------|---------------------|
| 前端发送 | chatStore.sendMessage 仅传 content；InlineChatInput 将引用展开进 content 后发送 | getReferences(tabId) → 构造 references 数组 → 与 content 一并传 ai_chat_stream |
| 后端接收 | ai_chat_stream 无 references 参数 | 增加 references 参数，转为 ReferenceInfo |
| 引用填充 | ai_commands 仅从 current_file 填充，messages 解析留空 | 前端 references 优先；current_file 仅作当前打开文件补充（不重复） |
| build_reference_prompt | 无 path 可信声明 | 含 File 时统一追加「路径已解析，可直接使用」 |

### 1.2 关键文件

| 层次 | 文件 | 改动点 |
|------|------|--------|
| 前端 | `chatStore.ts` | sendMessage 取 getReferences，构造 references，传 invoke |
| 前端 | `inlineContentParser.ts` | formatNodesForAI、parseEditorContent 与 references 协同；消息以标签展示 |
| 前端 | `referenceStore.ts` | 补充 buildReferencesForProtocol（Reference[] → 协议格式） |
| 后端 | `ai_commands.rs` | ai_chat_stream 增加 references 参数；用前端 references 构建 ContextInfo |
| 后端 | `context_manager.rs` | build_reference_prompt 含 File 时追加 path 可信声明 |

## 二、Phase 0：references 传递与 path 可信声明（P0）

**目标**：前端将 references 传给 ai_chat_stream；后端接收并注入；build_reference_prompt 含 File 时追加 path 可信声明。

### 步骤 0.1：前端协议转换

**文件**：`src/utils/referenceProtocolAdapter.ts`（新建）

**职责**：将 `Reference[]` 转为 ai_chat_stream 协议格式。

```typescript
// 输出格式（与设计文档 6.1 一致）
interface ReferenceProtocol {
  type: 'text' | 'file' | 'folder' | 'image' | 'table' | 'memory' | 'link' | 'chat' | 'kb' | 'template';
  source: string;
  content: string;
  editTarget?: { blockId: string; startOffset: number; endOffset: number };
  templateType?: 'document' | 'workflow' | 'skill';
}
```

**规则**：
- Text：source=sourceFile，content=完整文本，editTarget 含 blockId/startOffset/endOffset（若有）
- File：source=path，content=已有 content 或空；**当前打开文件不加入**；**用户 @ 当前打开文件忽略**
- Folder：source=path，content=loadFolderContent 结果（或占位）
- Memory：source=memoryId/name，content=items 拼接或 search_memories 结果；未实现时 `[记忆库功能未实现]`
- 其他类型：按设计文档 4.x 映射
- 大文件（>10MB）：仅传 path，content 为空，后续 prompt 提示 AI read_file

### 步骤 0.2：chatStore.sendMessage 传递 references

**文件**：`src/stores/chatStore.ts`

**改动**：
1. 在 invoke 前调用 `getReferences(tabId)`
2. 调用 `buildReferencesForProtocol(refs, currentFile)` 过滤并转换（排除当前打开文件、用户 @ 当前打开文件）
3. invoke 增加 `references: referencesProtocol`

```typescript
const { getReferences } = useReferenceStore.getState();
const refs = getReferences(tabId);
const references = await buildReferencesForProtocol(refs, currentFile);

await invoke('ai_chat_stream', {
  tabId,
  messages,
  modelConfig: {...},
  enableTools: enableTools,
  currentFile,
  selectedText,
  currentEditorContent,
  editTarget,
  references,  // 新增
});
```

### 步骤 0.3：InlineChatInput 与 ChatInput 协同

**发送路径**：InlineChatInput 与 ChatInput **均通过 chatStore.sendMessage** 发送。references 在 sendMessage 内统一从 `getReferences(tabId)` 取、经 `buildReferencesForProtocol` 转换后传 invoke，**不存在独立发送路径**。

**现状**：InlineChatInput 用 formatNodesForAI 将引用展开进 content；ChatInput 可能直接传 content。

**设计文档**：消息记录以**标签形式**展示，不展开；完整 content 通过 references 传给 AI。

**改动**：
- **发送时**：content 中引用以标签占位（如 `@文件名`）或保留简短占位；**实际引用内容仅通过 references 数组传递**
- 若当前 content 已含展开的引用文本，可保留（AI 仍能读），但**必须同时传 references**，保证 build_reference_prompt 有数据
- **消息展示**：用户消息中的引用以标签形式渲染，不展开（见 2.6）

**实现**：formatNodesForAI 或 sendMessage 前，content 仅作「用户输入文本+引用标签占位」；引用完整内容由 buildReferencesForProtocol 从 getReferences 取，写入 references。

### 步骤 0.4：后端 ai_chat_stream 接收 references

**文件**：`src-tauri/src/commands/ai_commands.rs`

**改动**：
1. `ai_chat_stream` 签名增加 `references: Option<Vec<ReferenceFromFrontend>>`
2. 定义 `ReferenceFromFrontend` 与前端协议一致：

```rust
// edit_target 必须为 Option，非 Text 类型引用无此字段，反序列化时避免 panic
#[derive(serde::Deserialize)]
struct ReferenceFromFrontend {
    #[serde(rename = "type")]
    reference_type: String,
    source: String,
    content: String,
    edit_target: Option<EditTargetInfo>,  // 仅 Text 类型有值，其他类型为 None
    template_type: Option<String>,
}
```
3. 若 `references` 非空，转为 `ReferenceInfo` 填入 `ContextInfo.references`
4. **设计文档为准**：用户 @ 当前打开文件 → 前端已过滤，不传。若前端 references 为空，仅将 current_file 作为引用（无引用场景的默认行为）。
5. 当前打开文件的内容由 build_context_prompt 第二层上下文传递，references 仅承载用户显式引用的资源

### 步骤 0.5：build_reference_prompt 追加 path 可信声明

**文件**：`src-tauri/src/services/context_manager.rs`

**改动**：
1. 在 `build_reference_prompt` 末尾，若 references 中含 `ReferenceType::File`，追加：
   ```
   The above file paths have been resolved and can be used directly. No need to call list_files or search_files.
   ```
2. 保持现有格式：`Reference N: Type (Source: source)\nContent: ...\n`
3. 末尾统一：`The above content has been fully provided. No need to read again.`

### 步骤 0.6：ReferenceInfo 与 ReferenceType 扩展

**文件**：`src-tauri/src/services/context_manager.rs`

**改动**：
- `ReferenceType` 增加 `Template`、`KnowledgeBase`（若尚未有）
- 确保与前端 type 枚举一一对应

## 三、Phase 1：降级逻辑、@ 列表、记忆库 content（P1）

### 步骤 1.1：降级为文本

**定义**：用户删除 @ 或引用标记后，引用降级为普通文本，不加入 references。

**文件**：`src/utils/inlineContentParser.ts`、`referenceProtocolAdapter.ts`

**逻辑**：
- `parseEditorContent` 或 `formatNodesForAI`：若 refId 对应标签的 textContent 不含 `@`，则按纯文本输出，不加入 refMap 或 references
- `buildReferencesForProtocol`：仅处理「有效引用」（标签含 @ 或 ref 有效）

### 步骤 1.2：@ 统一列表（五类树状、两条逻辑、激活规则）

**范围**：ChatInput、InlineChatInput 均需支持。

**组件**：新建或扩展 `MentionSelector`，支持：

| 能力 | 说明 |
|------|------|
| 五类 | 工作区文件、记忆库、知识库、模板库、聊天标签 |
| 点选 | @ 后展示五类树，点击展开懒加载 |
| 字符匹配 | 第一个非空格字符触发，跨五类匹配 |
| 激活规则 | 仅输入 @ 激活；空格取消；只激活一次；回删形成的 @ 不激活 |
| Tab | 作用于候选项 |
| 空匹配 | 展示空 |

**依赖**：知识库、模板库未实现时，@ 列表仅展示文件、记忆、聊天；可占位或灰显。

**1.2 实现与优化记录**：

| 优化项 | 说明 |
|--------|------|
| **@ 未触发修复** | InlineChatInput 使用 contentEditable，`handleInput` 增加 `[tabId]` 依赖，`requestAnimationFrame` 延迟检测确保 DOM/选区更新后再匹配；`compositionend` 时再次触发检测，解决 IME 输入法下 @ 不显示问题 |
| **位置** | 选择器由光标下方改为**光标上方**显示，`getCaretPosition` 返回光标顶部，MentionSelector 使用 `transform: translateY(calc(-100% - 4px))` 上浮并留 4px 间距 |
| **宽度约束** | 总宽度受聊天窗口限制，`getCaretPosition` 返回 `containerWidth`，`maxWidth = Math.max(200, containerWidth - left - 16)`，两种引用列表（点选模式、字符匹配模式）均适用 |
| **ChatInput 结构** | MentionSelector 作为 containerRef 直接子元素，containerRef 增加 `relative`，保证定位坐标系一致 |

### 步骤 1.3：记忆库 content 完整传入

**文件**：`referenceStore.formatForAI`、`inlineContentParser.formatReferenceForAI`、`referenceProtocolAdapter`

**逻辑**：
- Memory 引用：若有 items，用 items 拼接 content；否则调用 `search_memories`（需后端/前端接口），将结果填入 content
- 占位：记忆库未实现时，content 为 `[记忆库功能未实现，占位]`

### 步骤 1.4：引用层 per-type 约束

**文件**：`referenceProtocolAdapter.ts`

**规则**：
- File >10MB：仅传 path，content 空
- Memory/KB/Template：单条 content 上限（如 2000 字符，可配置）
- Folder：前 20 个文件预览

## 四、Phase 2：拖拽、Table、提示词构建层总预算（P2）

### 步骤 2.1：拖拽记忆/知识/模板/聊天

**前置**：各面板（记忆库、知识库、模板库、聊天标签）需支持 `draggable`，并设置 `dataTransfer` 含引用信息。

**文件**：`ChatInput.tsx`、`InlineChatInput.tsx`

**逻辑**：onDrop 时解析 dataTransfer，构造对应 Reference，addReference(tabId, ref)。

### 步骤 2.2：formatNodesForAI 增加 TABLE 分支

**文件**：`src/utils/inlineContentParser.ts`

**逻辑**：TableReference 时，将 tableData 格式化为表格文本或结构化描述传入。

### 步骤 2.3：提示词构建层总预算

**文件**：`context_manager.rs`

**逻辑**：扩展 TruncationStrategy 或新增 references 截断；references 与 messages 共享 token 预算；超限时按优先级裁剪（用户明确引用 > 自动注入）。

## 五、Phase 3：外部文件上传、知识库/模板库占位（P3）

### 步骤 3.1：外部文件拖拽上传

**逻辑**：`e.dataTransfer.files` 中 path 不在 workspace → 调用上传接口 → 写入缓存/临时存储 → 创建 FileReference（path 指向临时标识）。

**依赖**：需后端 `save_external_file` 或等效上传命令。

### 步骤 3.2：知识库/模板库占位

**逻辑**：references 含 kb/template 类型时，content 传 `[知识库功能未实现]` / `[模板库功能未实现]`，保证协议完整。

## 六、实施顺序与验收

### 6.1 实施顺序

```
Phase 0（P0）→ Phase 1（P1）→ Phase 2（P2）→ Phase 3（P3）
```

### 6.2 Phase 0 验收

| 项 | 验收标准 |
|----|----------|
| references 传递 | 前端 invoke 含 references；后端 ContextInfo.references 有数据 |
| **InlineChatInput 场景** | InlineChatInput 发送时 references 同样传入 ai_chat_stream，后端 ContextInfo.references 有数据（与 ChatInput 共用 sendMessage 路径，需分别验证） |
| **references 为空** | 用户未添加任何引用时，references 为空数组或不传；后端正常处理，current_file 作为默认上下文，不报错、不影响现有功能 |
| path 可信声明 | build_reference_prompt 含 File 时末尾有「路径已解析，可直接使用」 |
| 当前打开文件 | 用户 @ 当前打开文件不加入 references；current_file 由第二层处理 |
| 消息展示 | 消息记录内引用以标签形式展示（可逐步完善） |

### 6.3 冲突处理原则

- 旧逻辑「从 messages 解析引用」：**删除**，改为纯前端 references 传递
- 旧逻辑「current_file 自动加入 references」：**替换**为「前端 references 为主；references 为空时仅 current_file 作为引用」
- 旧逻辑 isOperationTarget：**不恢复**，path 可信由 prompt 统一声明
