## 一、触发条件与光标无效

### 1.1 须光标激活

- **有效条件**：编辑器聚焦时，Cmd+J 有效
- **无效条件**：光标未激活（编辑器未聚焦）时，快捷键无效
- **无效时行为**：窗口底部状态栏「保存状态处」滚动显示「指令无效」等提示，短时后清除

### 1.2 实现要点

- 在 Cmd+J 处理前判断 `editor.isFocused`（或等效）
- 未聚焦时：调用提示显示逻辑，`return` 不执行续写
- 与局部修改（Cmd+K）的光标无效处理一致

## 二、定位算法

### 2.1 基准

- 基于光标在文档中的 ProseMirror 位置（`from`）
- 使用 `editor.view.coordsAtPos(from)` 获取光标在视口中的 DOM 坐标

### 2.2 布局策略（明确优先级）

**参照物**：以**编辑窗口**（`.editor-zoom-scroll` 容器）为边界参照，悬浮卡不超出编辑区域，不覆盖聊天窗口等右侧面板。

悬浮卡定位按以下优先级与顺序决策：

| 优先级 | 场景 | 策略 |
|--------|------|------|
| **1. 首选** | 默认 | 悬浮卡出现在**光标右下方**：`top = 光标底部 + 8px`，`left = 光标左侧`，向右展开 |
| **2. 右侧空间不足** | `光标左侧 + 悬浮卡宽度 > 编辑窗口右边缘` | 悬浮卡**右边**紧贴编辑窗口右边缘：使用 `right` 定位，使悬浮卡右边缘 = 编辑窗口右边缘 - 8px（与宽度无关，避免向左偏移过多） |
| **3. 下方空间不足** | `光标底部 + 悬浮卡高度 > 编辑窗口下边缘` | 改为**右上方**：悬浮卡**底部** = 光标顶部 - 8px（与右下时「顶部 = 光标底部 + 8px」对称），使用 `bottom` 定位 |

**决策顺序**：先判断下方空间，再判断右侧空间；两者可同时生效（如光标在右下角时，既贴右又贴上）。

### 2.3 空间检测与边界约束

- **悬浮卡尺寸**：`min-w: 200px`，`max-w: 400px`，`max-h: 240px`；计算时按 `width=400`、`height=240` 预留空间
- **编辑窗口**：通过 `editor.view.dom` 向上查找 `.editor-zoom-scroll` 获取 `getBoundingClientRect()`
- **右侧空间**：`spaceRight = 编辑窗口.right - cursorLeft`；**不足时**（`spaceRight < 320px`）使用 `right` 定位，使悬浮卡右边缘 = 编辑窗口右边缘 - 8px（理论需 408px，留余量避免误判过早贴右）
- **下方空间**：`spaceBelow = 编辑窗口.bottom - cursorBottom`；**不足时**（`spaceBelow < 200px`）使用 `bottom` 定位，使悬浮卡底部 = 光标顶部 - 8px（理论需 248px，留余量避免误判过早改上方）
- **边界 clamp**：`left`、`top`/`bottom` 均限制在编辑窗口内

### 2.4 定位实现

- 使用 `position: fixed`，通过 **React Portal** 渲染到 `document.body`，避免受编辑器父级 `transform`、`overflow` 影响
- **不参与文档流**，零重排
- 基于 `editor.view.coordsAtPos(position)` 获取光标在视口中的坐标（`left`、`right`、`top`、`bottom`）

## 三、键盘导航

| 按键 | 行为 |
|------|------|
| **Tab** | 循环选中下一条（高亮当前项） |
| **Shift+Tab** | 循环选中上一条 |
| **Enter** | 应用当前选中项 |
| **Esc** | 关闭悬浮卡，不应用 |

**AI 一次返回 3 条**，Tab/Shift+Tab 切换选中，Enter 应用当前选中项，Esc 关闭。

## 四、关闭与重构逻辑

### 4.1 关闭

执行与悬浮卡无关的**任何操作**均直接关闭，包括：

- 手动输入
- 鼠标点击空白区域
- 保存
- 缩放窗口

### 4.2 重构

- 悬浮卡已显示时，再次按 Cmd+J → 直接重构悬浮卡（重新请求并展示新建议）
- 不关闭再打开，而是原地替换内容

## 六、续写长度

- **范围**：1-80 **字符**（此处「字」= 字符，即 character，含中文、英文、数字、标点等）
- **实现**：前端传 `maxLength: 80`，后端提示词与 `max_tokens` 据此限制
- **说明**：每条建议最多 80 字符，以完整句子或语义单元结尾

## 七、与 useAutoComplete 的集成

- useAutoComplete 改为返回 `{ suggestions: string[], selectedIndex }`，供悬浮卡消费
- 保留 AI 调用逻辑（ai_autocomplete）
- **AI 一次返回 3 条**，Tab/Shift+Tab 切换选中

---

## 七、与幽灵文字的关系

- **幽灵文字已放弃**：不再使用 GhostTextExtension，不保留回退
- 悬浮卡为辅助续写的**唯一**展示方式
