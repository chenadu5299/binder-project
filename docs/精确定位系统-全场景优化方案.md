# 精确定位系统：全场景优化方案

> 目标：使引用、选区、AI 自行检索等**所有编辑场景**均使用 blockId + offset 精确定位，消除基于 target_content 的检索逻辑。
>
> **详见**：`文本精准定位方案.md`（已整理当前实现与块+偏移方案）

---

## 一、编辑场景分类

| 场景 | 触发方式 | 当前定位方式 | 目标定位方式 |
|------|----------|--------------|--------------|
| **A. 选区发送** | 用户在编辑器中选中文本 → 点击发送 | ✅ 选区 → createAnchorFromSelection → edit_target(blockId+offset) | 已实现 |
| **B. 引用粘贴** | 用户从编辑器复制 → 粘贴到聊天 → 发送 | ✅ 复制时写入 blockId+offset → edit_target | 已实现 |
| **C. AI 自行编辑** | AI 根据上下文决定编辑，无用户选区/引用 | target_content + instruction 解析，全文 diff 或 replacen 第一处 | 待优化：prompt 注入 block_id 或禁用降级 |

## 三、数据流设计（目标状态）

### 3.1 场景 A：选区发送

```
用户选中 → editor.state.selection (from, to)
    → createAnchorFromSelection(doc, from, to)
    → findBlockContainingSelection + pmRangeToBlockOffset
    → edit_target = { anchor: { block_id, start_offset, end_offset } }
    → invoke('ai_chat_stream', { editTarget, ... })
    → ai_commands 注入 parsed_arguments["edit_target"]
    → tool_service: 若有 edit_target → extract_block_text_by_id + offset → 单条精准 diff
```

### 3.2 场景 B：引用粘贴

```
用户复制 → CopyReferenceExtension copy 事件
    → findBlockContainingSelection(doc, from, to)
    → pmRangeToBlockOffset(block, docStart, from, to)
    → sourceData = { blockId, startOffset, endOffset, ... }
    → 粘贴 → createTextReferenceFromClipboard(source) → TextReference 含 blockId+offset
    → 发送时 chatStore: ref.blockId && pathMatch → edit_target
    → 同场景 A 后端路径
```

### 3.3 场景 C：AI 自行编辑

**方案 C1（推荐）**：在 prompt 中注入「当前文档的块结构摘要」，要求 AI 在工具调用时输出 `block_id` 和 `start_offset`、`end_offset`。  
- 优点：完全精确  
- 缺点：需扩展工具 schema，AI 可能不稳定  

**方案 C2**：无 edit_target 时，**禁止**用 target_content 做 replacen 第一处；改为要求用户必须通过选区或引用提供位置。  
- 优点：逻辑简单，无歧义  
- 缺点：部分「自然语言描述位置」的请求无法满足  

**方案 C3（折中）**：保留 target_content 检索作为**最后降级**，但仅在**明确无任何位置信息**时使用，并打日志警告。同时优化检索逻辑（如 occurrenceIndex）减少错误。

## 六、验收标准

1. **场景 A**：选中编辑器文本发送，diff 精确落在选中区间，无错误位置。
2. **场景 B**：从编辑器复制 → 粘贴到聊天 → 发送，diff 精确落在复制区间。
3. **场景 C**：在无 edit_target 时，有明确降级策略，且不静默替换错误位置。
4. **日志**：每次工具调用能区分「Anchor 路径」与「降级路径」，便于排查。
