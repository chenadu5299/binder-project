## 一、项目概览

### 1.1 基本信息

| 项目 | 信息 |
|------|------|
| **项目名称** | Binder（合页） |
| **产品定位** | AI 智能办公套件，面向创作者的桌面端文档编辑器 |
| **应用类型** | 跨平台桌面应用（Tauri 2） |
| **当前版本** | 0.1.0 |

### 1.2 核心能力

- **文档编辑**：Markdown、HTML、DOCX、TXT 富文本编辑，TipTap + 分页
- **AI 辅助**：局部修改（Cmd+K）、对话编辑、工具调用、文档分析
- **本地资源**：工作区、文件树、全文搜索、记忆库
- **多格式预览**：DOCX、Excel、CSV、PDF、图片、演示文稿、音视频

#### 2.3.1 AI 编辑三种形式的提示词构建逻辑

| 形式 | 触发 | 前端收集 | 后端构建 | 实现位置 |
|------|------|----------|----------|----------|
| **层次一 辅助续写** | Cmd+J | context_before（500 字符）、context_after（200 字符）、position、documentFormat | build_autocomplete_prompt | useAutoComplete → ai_autocomplete → deepseek.rs |
| **层次二 局部修改** | Cmd+K | instruction、text、context（前后各 500 字符）、messages（多轮） | 硬编码 system + user prompt | useInlineAssist → ai_inline_assist → deepseek.rs inline_assist |
| **层次三 对话编辑** | 聊天窗口 | messages、current_file、selected_text、references | build_multi_layer_prompt | chatStore → ai_chat_stream → context_manager.rs |

---

**层次一 辅助续写**（`ai_autocomplete`）

- **构建函数**：`deepseek.rs::build_autocomplete_prompt`
- **System Prompt**：角色（写作助手）+ 文档格式要求（t-docx/md/html/纯文本）+ 续写规则（长度、全文理解、插入位置、时间线约束、上下文衔接）
- **User Prompt 构建顺序**：
  1. `[文档概览]`（可选）：文档长度、当前章节、结构、开头/结尾、前后段落
  2. `[上下文内容]`：上文（≤600 字符，句子边界截断）、下文（≤400 字符或无）
  3. `[结构信息]`（可选）：node_type、heading_level、list_type、list_level、block_type
  4. `[记忆库信息]`（可选）：entity_name、entity_type、content（≤100 字符）

5. `[续写要求]`：插入位置说明、时间线约束、输出格式（3 条用 `---` 分隔）

- **输出格式**：3 条续写建议，用 `---` 分隔

---

**层次二 局部修改**（`ai_inline_assist`）

- **构建方式**：`deepseek.rs::inline_assist` 内硬编码
- **System Prompt**：角色（文档和内容处理助手）+ 多用途能力（文本修改、内容生成、分析讨论、分类匹配）
- **User Prompt 构建顺序**：
  1. `[用户指令]`：用户输入的 instruction
  2. `[选中文本]`：选中的文本或「无选区」
  3. `[上下文内容]`：前端拼接的 `【上下文（前）】\n${contextBefore}\n\n【选中文本】\n...\n\n【上下文（后）】\n${contextAfter}`；若有 messages，后端在 context 前拼接 `【历史对话】\nuser: xxx\nassistant: xxx\n\n`
  4. `[任务要求]`：理解指令意图（修改/生成/分析/分类等）、按类型输出
  5. `[输出格式要求]`：JSON `{ kind: "edit"|"reply", text: "..." }`

- **输出格式**：JSON，`kind` 为 `edit`（修改文本）或 `reply`（分析/说明）

---

**层次三 对话编辑**（`ai_chat_stream`）

- **构建函数**：`context_manager.rs::build_multi_layer_prompt`
- **四层结构**：
  1. **第一层 基础系统提示词**：`build_base_system_prompt(enable_tools)`
     - Agent 模式：完整英文版（意图识别、决策原则、执行原则、工具调用、禁止项等）
     - Chat 模式：精简版「You are an expert document assistant」
  2. **第二层 上下文提示词**：`build_context_prompt`

     - 当前文档：`current_file`、编辑规则（`edit_current_editor_document`；优先 `edit_mode`+`scope` 编辑操作层，禁用对当前文件先 `read_file` 再编辑）
     - 选中文本：`selected_text`
     - 工作区路径、编辑器状态、未保存提示

  3. **第三层 引用提示词**：`build_reference_prompt`（详见《引用功能完整设计文档》）

     - 用户引用内容（Text/File/Folder/Image/Chat/Link/Template 等）；references 由前端传递
     - 含 File 时统一追加「路径已解析，可直接使用」；消息以标签形式展示，不展开

  4. **第四层 工具调用**：Agent 模式下工具格式要求已包含在第一层

- **消息格式**：系统消息（上述 prompt）+ 用户消息列表；引用内容写入消息中
