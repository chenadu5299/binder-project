## 一、项目概览

### 1.1 基本信息

| 项目 | 信息 |
|------|------|
| **项目名称** | Binder（合页） |
| **产品定位** | AI 智能办公套件，面向创作者的桌面端文档编辑器 |
| **应用类型** | 跨平台桌面应用（Tauri 2） |
| **当前版本** | 0.1.0 |
| **产品描述** | 面向创作者的 AI 智能办公套件，提供文档、表格、演示文稿三大核心能力，配备全面的 AI 辅助功能和完善的本地资源管理系统 |

### 1.2 核心价值

- **文档编辑**：支持 Markdown、HTML、DOCX、TXT 等多格式，基于 TipTap 的富文本编辑
- **AI 辅助**：三层 AI 能力（自动补全快捷键触发、内联辅助 Cmd+K、聊天窗口），支持工具调用与文档分析
- **本地资源管理**：工作区、文件树、搜索索引、记忆库一体化管理
- **多格式预览**：DOCX、Excel、演示文稿、CSV、PDF、图片等一站式预览

## 四、功能逻辑

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    前端 (React + Vite)                                        │
│  ┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────────────────────┐ │
│  │  Layout    │ │  Editor   │ │   Chat    │ │  FileTree / Memory / Search │ │
│  │  MainLayout│ │  TipTap   │ │  AI 对话  │ │  资源管理区                  │ │
│  └────────────┘ └────────────┘ └────────────┘ └────────────────────────────┘ │
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │  Zustand Stores: chatStore, editorStore, fileStore, layoutStore, ... │ │
│  └─────────────────────────────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────────────────────────────┐│
│  │  Services: documentService, fileService, memoryService, ...             ││
│  └─────────────────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ invoke()
┌───────────────────────────────────▼─────────────────────────────────────────┐
│                     Tauri 2 桥接层 (IPC)                                       │
└───────────────────────────────────┬─────────────────────────────────────────┘
                                    │
┌───────────────────────────────────▼─────────────────────────────────────────┐
│                     Rust 后端                                                 │
│  Commands → Services → 外部工具 (Pandoc, LibreOffice)                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 核心数据流

```
1. 文档编辑：用户操作 → TipTap → editorStore → documentService → Tauri write_file / save_docx
2. AI 对话：用户输入 → chatStore → ai_chat_stream → AIService → 流式返回 → ChatMessages
3. 文件树：工作区变更 → build_file_tree → fileStore → FileTreePanel
4. 文件监听：notify → FileWatcherService → check_external_modification → 前端刷新
```

### 4.3 功能模块详细列表

#### 4.3.1 文件与工作区

- 工作区管理：打开、切换、加载、监听
- 文件树：构建、打开、增删改移、拖拽、元数据
- 文件监听：外部修改检测（`check_external_modification`）
- 文件分类：`classify_files`、`organize_files`
- 文件元数据：记录 Binder 创建的文件来源（new、ai_generated、external）

#### 4.3.2 文档编辑

- 富文本：TipTap 富文本、多标签、自动保存（2 秒防抖）
- 格式：DOCX、Markdown、HTML、TXT
- 多格式预览：DOCX/Excel/演示文稿/CSV/PDF/图片等
- 图片：插入、base64、压缩（WebP）、相对路径

#### 4.3.3 AI 三层架构

| 层次 | 功能 | 触发方式 | 状态 |
|------|------|----------|------|
| **层次一** | 自动补全（幽灵文字） | 快捷键触发 | 95% |
| **层次二** | 内联辅助（Diff 视图） | Cmd+K 快捷键 | 100% |
| **层次三** | AI 聊天窗口（流式、工具调用、文档引用） | 手动打开窗口 | 100% |

- 多模型：DeepSeek、OpenAI
- 工具调用：`execute_tool`、`execute_tool_with_retry`
- 文档分析：`ai_analyze_document`
- API Key：`ai_save_api_key`、`ai_get_api_key`

#### 4.3.4 搜索与记忆

- 全文搜索：SQLite FTS5 索引、`search_documents`、`index_document`、`build_index_async`
- 记忆库：`add_memory`、`get_document_memories`、`search_memories`、`delete_memory`、`check_memory_consistency`

#### 4.3.5 资源管理区域

- 工作区：文件树展示
- 记忆库：从 AI 聊天窗口迁移
- 历史记录：记录用户操作（最多 50 步）
- 顶部工具栏：新建文件夹、新建文件、搜索

## 九、构建与运行

### 9.1 环境要求

- Node.js ≥ 18
- Rust ≥ 1.70（stable）
- Pandoc（可执行文件）
- LibreOffice（可选，用于 DOCX/Excel/演示文稿预览）

### 9.2 开发命令

```bash
# 安装依赖
npm install

# 开发模式（前端 + Tauri）
npm run tauri:dev

# 仅前端开发
npm run dev

# 构建（含 tiptap-pagination-plus 本地依赖构建）
npm run build
npm run tauri:build
```

## 十一、总结

Binder（合页）是一个约 **5 万行代码** 的 Tauri 2 桌面应用，采用 **React + TypeScript** 前端和 **Rust** 后端，围绕文档编辑、AI 辅助、文件管理和记忆库构建。架构清晰，前后端分离，功能模块完整，但存在较多 TypeScript 类型和构建问题，需要优先修复以稳定构建和开发体验。
