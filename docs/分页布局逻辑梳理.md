# 分页布局逻辑梳理

> 本文档描述 T-DOCX 分页模式下的容器层级、overflow 策略与当前页检测逻辑。功能概览见 `分页功能说明.md`。

---

## 一、名词与单位

| 名词 | 含义 | 单位 | 来源 |
|------|------|------|------|
| `--rm-page-width` | 纸张总宽度 | px | updateCssVariables |
| `--rm-page-height` | 纸张总高度 | px | updateCssVariables |
| `--rm-margin-left` | 左边距 | px | updateCssVariables |
| `--rm-margin-right` | 右边距 | px | updateCssVariables |
| 内容区宽度 | 可编辑文字区域 | = pageWidth - marginLeft - marginRight | 计算得出 |

## 二、布局链（完整容器层级）

```
MainLayout: flex-1 min-w-0 overflow-hidden
└── EditorPanel: overflow-hidden minWidth: 0
    └── 编辑器内容区: flex-1 overflow-hidden flex minWidth: 0
        └── 编辑器滚动区 (.editor-zoom-scroll): flex-1 overflow-auto minWidth: 0
            │   ← 分页模式下纵向+横向滚动统一发生在此，避免与 EditorContent 产生重复/重合滚动条
            └── Zoom 包装: width=794*(zoom/100) margin:0 auto
                │   ← 显式 width 使 scrollWidth 正确；zoom≠100 时内层 transform scale 实现视觉缩放
                └── TipTapEditor 包装 div: overflow-visible min-w-[794px]
                    └── EditorContent: flex-1 py-4 px-0 overflow-visible min-h-full
                        │   ← 分页模式下始终 overflow-visible，滚动由 editor-zoom-scroll 统一处理
                        └── ProseMirror (.rm-with-pagination)
                            ├── width: var(--rm-page-width)
                            ├── paddingLeft/Right: var(--rm-margin-*)
                            └── overflow-x: visible（data-zoomed 时由 CSS 覆盖，由 editor-zoom-scroll 统一处理横向滚动）
```

## 三、文字遮挡问题与容器方案

### 3.1 问题根因

纸张宽度固定（如 794px），而编辑窗口宽度可变（约 400–800px）。若中间任一容器使用 `overflow-hidden`，会将纸张裁剪到父级宽度，导致右侧文字被遮挡。

### 3.2 最终容器方案（适用）

| 层级 | 容器 | overflow | 说明 |
|------|------|----------|------|
| EditorPanel 包装 | `div.h-full.overflow-visible` | `overflow-visible` | 不能裁剪纸张 |
| TipTapEditor 包装 | `div.h-full.overflow-visible` | `overflow-visible` | 不能裁剪纸张 |
| EditorContent | 分页时 `py-4 px-0`，无水平 padding | - | 避免与页边距叠加，页边距为 0 时文字不被遮挡 |
| 编辑器滚动区 | `.editor-zoom-scroll` | `overflow-auto` | 由上层提供横向/纵向滚动 |

### 3.3 页边距与 padding

- 分页模式下，EditorContent 使用 `py-4 px-0`，无水平 padding
- 页边距由 PaginationPlus 的 `--rm-margin-*`、`--rm-content-margin-*` 控制，与 CSS 变量同步
- 若 EditorContent 有水平 padding，会与页边距叠加，页边距为 0 时文字仍被遮挡
- **光标遮罩**：跨页长光标的遮罩高度需覆盖 contentMargin、margin、页眉页脚，详见 `docs/分页模式光标与遮罩完整方案.md`
- **模拟光标**：PageTopCaretExtension 挂载到 editor-zoom-scroll，依赖 `transform: scale` 与当前 overflow 策略。若调整缩放实现或滚动容器，需参考 `docs/分页模式光标与遮罩完整方案.md` 第 3.7 节「影响光标方案效果的改动清单」

## 四、滚动容器与当前页检测

### 4.1 滚动发生位置

分页模式下，**所有滚动统一由 `editor-zoom-scroll` 处理**，无论缩放比例：

| 状态 | 滚动容器 | 说明 |
|------|----------|------|
| 分页模式（任意缩放） | `editor-zoom-scroll` | `scrollOnEditor=false`，EditorContent 为 `overflow-visible`，横向+纵向滚动均由 editor-zoom-scroll 统一处理，避免重复/重合滚动条 |

**缩放实现**：zoom≠100% 时，外层 width=794*(zoom/100) 使 scrollWidth 正确；内层 `transform: scale()` 实现视觉缩放。`margin: 0 auto` 实现内容横向居中。

`usePaginationFromEditor` 查找滚动容器时，从 `editor.view.dom` 向上遍历，**返回第一个** `overflow-y: auto/scroll` 的祖先（分页模式下为 editor-zoom-scroll）。

**缩放实现说明**：当前使用 `transform: scale()` 实现视觉缩放（非 CSS zoom），T-DOCX 时在 width 包装内另有 transform 包装。Firefox 对 zoom 支持有限，项目未强制兼容。

### 4.2 当前页检测逻辑

- **数据来源**：PaginationPlus 的 `[data-rm-pagination]` 容器下的 `.rm-page-break` 元素
- **算法**：取滚动容器视口中心 Y 坐标，遍历各 `.rm-page-break` 的 `getBoundingClientRect()`：
  - 若中心落在某页 `[top, bottom]` 内 → 当前页 = 该页
  - 若中心在某页 top 之上 → 当前页 = 上一页（视口中心在两页之间的分割缝时，视为正在浏览上一页）
- **触发**：监听滚动容器的 `scroll` 事件，使用 `requestAnimationFrame` 更新；分页结构变化（`totalPages` 更新）时重新绑定

### 4.3 工具栏页码导航

- **显示条件**：`fileType === 'docx'` 且 `totalPages > 1`
- **内容**：上一页 / 当前页/总页数 / 下一页
- **实现**：`EditorToolbar` 使用 `usePaginationFromEditor(editor, fileType === 'docx')` 获取 `currentPage`、`totalPages`、`scrollToPage`
