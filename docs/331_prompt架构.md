# 331_prompt架构

## 一、文档定位

本文是 AI 执行系统中提示词架构的统一主规范，收口三层能力的提示词构建链、职责边界与跨层映射关系。  
本文用于替代分散口径，不替代各层专项提示词细化文档。

---

## 二、MVP 目标

1. 明确三层提示词的构建路径与执行边界。  
2. 建立前端采集、后端拼接、模型调用、结果回流的统一映射。  
3. 统一变更治理，避免提示词升级导致跨层行为漂移。  

---

## 三、能力范围

1. 覆盖层次一（辅助续写）、层次二（局部修改）、层次三（对话编辑）。  
2. 覆盖提示词构建链与工具定义注入链。  
3. 覆盖提示词版本治理与回归校验入口。  

不在本文范围：
1. 各层提示词逐条文案细节。  
2. 模型厂商参数细节。  

---

## 四、统一分层架构

## 4.1 三层能力映射

| 层次 | 能力 | 触发入口 | 提示词构建主位 |
|---|---|---|---|
| 层次一 | 辅助续写 | 编辑器快捷触发 | 前端采集 + 后端拼接 |
| 层次二 | 局部修改 | Inline Assist 触发 | 前端采集 + 后端拼接 |
| 层次三 | 对话编辑 | 聊天消息触发 | ContextManager 多层构建 |

## 4.2 统一构建链

1. 上下文采集层  
按层次采集文档上下文、选区、引用、会话信息。  

2. 提示词组装层  
按层次规则组装系统提示、上下文提示、引用提示、工具提示。  

3. 执行层  
调用 AI Provider，输出文本或工具调用。  

4. 回流层  
将结果写回编辑器、对话流、Diff 链或状态链。  

---

## 五、跨层映射关系（核心补齐）

## 5.1 层次一映射

1. 采集入口：`useAutoComplete.ts`。  
2. 后端入口：`ai_commands.rs::ai_autocomplete`。  
3. 输入要素：上下文片段、光标位置、文档格式。  
4. 输出形态：短续写文本（直接反馈编辑态）。  

## 5.2 层次二映射

1. 采集入口：`useInlineAssist.ts`。  
2. 后端入口：`ai_commands.rs::ai_inline_assist`。  
3. 输入要素：instruction、selectedText、局部上下文、可选 messages。  
4. 输出形态：局部替换文本（直接作用编辑区）。  

## 5.3 层次三映射

1. 采集入口：聊天发送链（chatStore/ChatPanel）。  
2. 后端入口：`ai_commands.rs::chat_stream`。  
3. 构建主位：`context_manager.rs`（系统层 + 上下文层 + 引用层 + 工具层）。  
4. 输出形态：流式消息 + 可选工具调用结果。  

## 5.4 工具定义映射

1. 定义来源：`tool_definitions.rs`。  
2. 注入方式：随对话执行链注入 Provider。  
3. 约束：工具定义版本必须与提示词规则同步。  

---

## 六、职责边界

1. Hook 层负责采集，不负责定义最终提示词规则。  
2. 后端命令层负责拼接与执行，不负责编辑器状态渲染。  
3. ContextManager 仅主导层次三，不越界到层次一/二。  
4. 工具定义层负责 schema，不承载业务文案逻辑。  

---

## 七、提示词治理规则

1. 单层变更必须登记影响层与影响链。  
2. 系统提示词与工具规则发生冲突时，以工具可执行约束优先。  
3. 同一能力仅保留一个“权威提示词规则入口”。  
4. 变更后必须执行三层回归校验。  

---

## 八、执行与回归口径

1. 层次一：校验触发稳定性、响应时延、文本风格一致性。  
2. 层次二：校验局部编辑准确性、上下文一致性、无越界修改。  
3. 层次三：校验多层提示组装、工具调用成功率、结果回流一致性。  
4. 跨层：校验共享术语、字段命名、错误暴露口径一致。  

---

## 九、与其他模块协作边界

1. 与 `204_ai执行架构`：331 定义提示词子架构，204 定义整体执行架构。  
2. 与 `332_工具调用体系`：331 定义工具提示词协作口径，332 定义工具运行时契约。  
3. 与 `334_分层提示词分析`：334 作为问题与修正细化，331 作为主规则入口。  
4. 与 `311/314`：对话编辑提示词必须与对话编辑业务规则同步。
