## 一、文档定位

本文是 Workspace 系统中资源管理能力的统一主规范，收口资源区域的对象模型、操作契约、状态规则与技术约束。  
本文承接需求文档功能面，补齐可落地实现约束。

本文不是 Workspace 工作台协同文档。  
凡涉及“顶部工具栏 / 主内容区 / 底部状态栏 / 资源分区如何共同构成工作台对象，以及这些入口如何与 AI 协同”的问题，以 `A-WS-M-D-01_workspace工作台协同主控文档.md` 为准。

---

## 二、MVP 目标

1. 建立资源管理区统一对象模型与操作主链。  
2. 建立资源区（工作区/记忆库/知识库/模板库/时间轴）一致的状态语义。  
3. 建立资源操作的技术约束（权限、性能、一致性、观测）。  

---

## 三、资源域对象模型

## 3.1 资源域对象

1. `WorkspaceResource`：工作区文件树资源（文件/目录节点）。  
2. `MemoryResource`：记忆项资源（实体、内容、来源、时间）。  
3. `KnowledgeResource`：知识库资源（知识库、分类、条目、来源文档）。  
4. `TemplateResource`：模板资源（工作流模板）。  
5. `TimelineResource`：时间轴节点资源（操作类型、作用对象、时间、摘要、可还原性）。  

## 3.2 资源标识

1. 每类资源必须有稳定主键（路径或 ID）。  
2. 资源显示名与资源标识分离，避免重命名导致引用失效。  
3. 资源类型必须可枚举，不允许临时字符串扩展。  

---

## 四、资源管理区主链

## 4.1 展示链

1. 顶部工具栏驱动资源操作入口。  
2. 主内容区以分区面板承载资源列表。  
3. 底部状态栏展示资源上下文与系统状态。  

## 4.2 操作链

1. 用户操作 -> 命令分发 -> 后端执行 -> 事件回流 -> 资源树刷新。  
2. 资源操作必须具备成功/失败反馈，不允许静默失败。  
3. 同类操作（创建、删除、重命名、移动）走统一事件协议。  

## 4.3 同步链

1. 文件系统监听驱动资源更新。  
2. AI 工具操作完成后触发资源刷新事件。  
3. 时间轴按已成立的项目逻辑状态变更落库并受容量策略约束。  
4. 时间轴生成采用“基于成立点收口、并以实际状态变化过滤”为准的策略，不按成立入口调用次数机械追加。  

---

## 五、技术实现约束（核心补齐）

## 5.1 操作权限约束

1. 所有资源操作必须在当前 workspace 边界内执行。  
2. 危险操作（删除、批量改名）必须走确认门禁。  
3. 非法路径、越界路径、无权限路径必须统一拒绝。  

## 5.2 一致性约束

1. 资源树显示状态与真实文件状态需最终一致。  
2. 操作成功后必须同步更新缓存与索引。  
3. 操作失败不得污染资源状态。  
4. 创建型操作必须产出目标格式的合法文件，不得以零字节文件或占位文本伪装成功。  

## 5.3 性能约束

1. 资源区刷新必须支持增量更新优先。  
2. 大目录展开需要分段加载或延迟渲染策略。  
3. 搜索与筛选优先走索引层，不做全量阻塞扫描。  

## 5.4 观测约束

1. 资源操作必须记录操作类型、目标、结果、耗时。  
2. 异常必须结构化输出（错误码/阶段/对象）。  
3. 关键操作链应可回放（至少日志级可追踪）。

## 六、资源分区规则

## 6.1 工作区分区

1. 以文件树为主，支持层级展开与基础文件操作。  
2. 支持搜索、重命名、移动、删除等操作入口。  
3. 与 `A-WS-M-T-03_文件系统.md` 文件系统共享路径与状态语义。  

## 6.2 记忆库分区

1. 记忆项在资源区是可检索、可浏览、可管理对象。  
2. 记忆项操作必须与记忆系统主链一致。  
3. 记忆项来源、关联对象必须可追溯。  

## 6.3 知识库分区

1. 知识库在资源区是用户主导的知识资产入口。  
2. 资源区可提供浏览、搜索、选择、存入、删除、替换入口。  
3. 知识库的注入优先级、自动检索语义与写入权限，以 ``A-AST-M-D-02_Binder Agent知识库协同主控文档.md`/`A-AST-M-T-02_知识库机制.md`` 为准。  

## 6.4 模板库分区

1. 模板库在资源区是约束资产入口，而不是内容仓。  
2. 资源区可提供模板浏览、选择、创建、保存、删除入口。  
3. 工作流模板的资产语义，以 ``A-TMP-M-T-01_模板机制.md`` / ``A-TMP-M-D-01_Binder Agent模板协同主控文档.md`` 为准；文档生成主链以 `A-AG-M-T-05_文档生成流程.md` 为准。

## 6.5 时间轴分区

1. 时间轴按时间倒序展示时间轴节点。  
2. 支持容量上限与过期淘汰策略。  
3. 时间轴是项目逻辑状态演进视图，不等于审计系统。  
4. 时间轴节点来源于成立点收口后的稳定状态变化，不展示未形成实际状态变化的空成立动作。

1. 与 `A-WS-M-T-03_文件系统.md`：`A-WS-M-T-03_文件系统.md` 负责文件主链，`A-WS-M-T-04_资源管理.md` 负责资源区编排与资源视图管理。  
2. 与 `A-WS-M-T-05_跨文档操作.md`：`A-WS-M-T-05_跨文档操作.md` 处理跨文档引用与任务流，`A-WS-M-T-04_资源管理.md` 提供资源检索与选择入口。  
3. 与 `A-WS-M-D-01_workspace工作台协同主控文档.md`：`A-WS-M-D-01_workspace工作台协同主控文档.md` 负责工作台级区域与入口协同，`A-WS-M-T-04_资源管理.md` 负责资源对象、资源操作与资源状态。  
4. 与 ``A-AST-M-T-01_记忆模型.md`/`A-AST-M-P-01_上下文注入.md`/`A-AST-M-D-01_Binder Agent记忆协同主控文档.md``：记忆相关资源受记忆系统主规则约束。  
5. 与 ``A-AST-M-D-02_Binder Agent知识库协同主控文档.md`/`A-AST-M-T-02_知识库机制.md``：知识库资源的资产语义与检索协同由知识库系统主控。

7. 与 UI 系统：UI 可调整展示，不可更改资源语义与状态规则。

## 八、MVP 验收口径

1. 资源区能够稳定承接工作区、记忆库、知识库、模板库、时间轴五类资源入口。  
2. 资源主操作链可执行且结果可观测。  
3. 资源状态与后端真实状态一致，无长期漂移。  
4. 性能与错误处理满足资源区稳定运行要求。
