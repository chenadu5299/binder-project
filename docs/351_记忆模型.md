# 351_记忆模型

## 一、文档定位

本文是记忆上下文系统中的记忆模型统一主规范，收口记忆分层、作用域、数据模型、生命周期与检索协议。  
本文承接需求锚定与主控口径，补齐模型定义层并增强与全局架构映射。

---

## 二、MVP 目标

1. 建立可执行的四层记忆模型（标签级、项目内容、工作区长期、用户级）。  
2. 建立记忆写入、更新、删除、检索的统一语义。  
3. 建立记忆模型与 AI 执行架构、Workspace 架构、引用系统的映射关系。  

---

## 三、模型总览

## 3.1 记忆层级

1. 标签级记忆（`tab`）  
作用域：单聊天标签；生命周期：随标签删除。  

2. 项目内容记忆（`content`）  
作用域：工作区项目语义层；生命周期：随文档变化更新。  

3. 工作区长期记忆（`workspace_long_term`）  
作用域：工作区；生命周期：只追加不回写修改。  

4. 用户级记忆（`user`）  
作用域：用户；生命周期：跨项目继承并演化。  

## 3.2 设计原则

1. 显式引用优先于自动检索。  
2. 作用域隔离优先于召回规模。  
3. 先结构化存储，再语义增强。  
4. 记忆写入失败不阻塞主对话。  

---

## 四、核心数据模型

## 4.1 记忆项（MemoryItem）

最小字段集合：
1. `id`  
2. `layer`  
3. `scope_type`  
4. `scope_id`  
5. `entity_type`  
6. `entity_name`  
7. `content`  
8. `summary`  
9. `source_kind`  
10. `source_ref`  
11. `confidence`  
12. `freshness_status`  
13. `readonly`  
14. `created_at`  
15. `updated_at`

## 4.2 来源模型（MemorySource）

冻结来源类型：
1. `conversation_summary`  
2. `conversation_entity`  
3. `document_extract`  
4. `document_outline`  
5. `document_detail_enrichment`  
6. `tab_deletion_summary`  
7. `user_preference`

## 4.3 关系模型（MemoryEdge）

1. 记忆项之间可建立引用/归属/演化关系。  
2. 关系必须可追溯来源，不允许隐式推断直接落库。  

---

## 五、生命周期模型

## 5.1 生成链

1. 对话触发 -> 标签级记忆抽取。  
2. 文档触发 -> 项目内容记忆抽取。  
3. 标签删除 -> 删除行为摘要写入长期记忆。  
4. 跨项目行为 -> 用户级记忆增量演化。  

## 5.2 更新链

1. 标签级记忆：仅在标签存续期内更新。  
2. 项目内容记忆：随文档结构与内容变化更新。  
3. 长期记忆：只追加，不回写修改。  
4. 用户级记忆：可演化，不受项目切换影响。  

## 5.3 删除链

1. 删除标签 -> 标签级记忆删除。  
2. 删除标签不会回写删除长期记忆历史。  
3. 工作区切换不会携带前工作区标签/项目记忆。  

---

## 六、检索协议

## 6.1 检索顺序

`显式引用 > 标签级记忆 > 项目内容记忆 > 长期记忆 > 用户级记忆`

## 6.2 检索范围

`tab | content | workspace_long_term | user | all`

## 6.3 检索规则

1. 先作用域过滤，再相关性排序。  
2. 同层优先最近更新、置信度高、来源明确项。  
3. 检索失败或超时按空结果降级，不阻塞主链。  

---

## 七、与全局架构映射（核心补齐）

1. 与 `202_workspace架构`  
记忆作用域与 workspace 边界绑定，路径与工作区上下文统一。  

2. 与 `203_数据流状态流`  
记忆写入、检索、注入属于统一状态流事件。  

3. 与 `204_ai执行架构`  
记忆检索结果进入提示词构建链，作为上下文增强层。  

4. 与 `331_prompt架构` / `352_上下文注入`  
记忆模型定义“有什么”，注入文档定义“何时、按何优先级注入”。  

5. 与 `343_跨文档操作`  
跨文档任务可消费记忆项，但不得越过作用域边界。  

---

## 八、约束与非目标

1. 记忆库不是用户可自由编辑的笔记系统。  
2. 首版不引入外部向量数据库作为主存。  
3. 不在层次一/层次二默认接入记忆模型。  
4. 不以召回数量替代作用域正确性。  

---

## 九、MVP 验收口径

1. 四层记忆对象可区分、可检索、可追溯。  
2. 标签删除、项目切换、跨项目继承行为符合模型定义。  
3. 检索顺序、范围、降级规则可复现。  
4. 与 202/203/204/331/352 文档口径一致。
